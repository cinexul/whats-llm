普段あなたが使う AI は、ほとんどが「よその家」に住んでいます――あなたが打った文字はネットワークを飛んで、どこかの会社のサーバーで計算され、答えが戻ってきます。この仕組みはとても手軽ですが、一部の人にとっては落ち着かない三つのことももたらします：あなたの内容は処理されるために **自分のパソコンを離れ** なければならない、**ネットが切れると使えない**、そしてよく使うと **使った分だけお金を払う** ことになる。

そこで、別の道を考え始める人が出てきました：モデルを **直接自分のパソコンに入れて**、ローカルで動かし、ネットにつながず、外に出さず、課金されないようにできないか、と。答えはできます――これが本章で扱う **ローカルモデル（local model）**、そして普通の人がそれを動かすのを助ける二種類のツール、Ollama と LM Studio です。先に一つはっきりさせておきます：ローカルモデルは「より高級な版」ではなく、「動かす場所を変えた版」です。それには甘い汁もあれば、実打実の代償もあります。

## 一、なぜ自分のパソコンでモデルを動かしたい人がいるのか

モデルを自宅に持ち帰るのは、三つのものを狙ってのことです。まずそれらがいったい何を解決するのかをはっきり見ましょう。

| 欲しいもの | それが何を解決したか | 誰が特に気にするか |
| --- | --- | --- |
| **秘匿性** | 内容が終始ローカルを出ず、どの会社のサーバーにもアップロードされない | 機微な資料を扱う、データを「外に出す」のが嫌な人 |
| **オフライン** | ネットがなくても使える――飛行機の中、社内ネット、ネットが切れても動く | ネットが制限される、または隔離環境で作業する必要がある人 |
| **回数で課金されない** | 入れた後は使い放題で、一回ごとに課金されない | 使用量が多い、または課金の心理的負担に引きずられたくない人 |

この三つの共通点は：どれも同じ一つのことから来るということです――**モデルがあなた自身のマシン上で動き、入力も出力もローカルを離れない**。クラウドのあの仕組みが手軽なのは、まさに重い仕事を「よその家」でやるからです。そしてローカルのこの仕組みが秘匿・オフライン・無課金なのも、まさに重い仕事を「自分の家」に持ち帰ったからです。**利点と代償は、同じ一つの原因の裏表** です。

> **ポイント:** 「ローカル」を一種の技自慢だと思わないでください。それは本質的に **トレードオフの問題** です。あなたは「自分のマシンの計算力と、少しの設定の手間」で、「秘匿・オフライン・無課金」を手に入れます。自分がどちらを最も気にするかをよく考えてから、この道を行くか決めましょう――「自分のパソコンに入れられる」というだけで人より上だと思うのではなく。

## 二、ローカル vs クラウド：一枚の表でトレードオフを整理する

これは本章で最も覚えるべき一枚の表です。ローカルとクラウドは「どちらが優れているか」ではなく、「異なる人、異なる場面のために用意されたもの」です。

| | クラウド(普段使うあの種類) | ローカル(自分のパソコンに入れる) |
| --- | --- | --- |
| データはどこへ | 会社のサーバーに送って計算 | **ローカルを出ない** |
| ネット接続が要るか | 要る | **不要** |
| お金の払い方 | 従量課金 / プランの利用枠 | 入れた後は **回数で課金されない**(ただし相応のパソコンが要る) |
| 能力の強弱 | 通常 **より強い**(背後は巨大なマシン) | **あなたのマシンに制限され**、同条件ではクラウドの最上位に **及ばないことが多い** |
| 始めやすさ | 開けば使えて手軽 | ダウンロード・設定が要り、少し敷居がある |
| 保守 | 会社が代わりに背負う | 自分で管理する |

一言で両者の性格をまとめると：

- **クラウド ＝ 強力・手軽だが、データが外に出る・従量でお金がかかる。**
- **ローカル ＝ 秘匿・オフライン・無課金だが、マシンに制限される・設定の敷居がある・通常クラウドほど強くない。**

最も見落とされやすい一行――**能力の強弱** に注意してください。クラウドで動くのはたいてい非常に大きなモデルで、背後には普通の人には買えない計算力があります。あなたのパソコンで動かせるのは、たいていその「スリム版」です。だから「ローカルモデルを入れれば、効果が最強のクラウドと同じ」と期待すると、おおかた失望します。これは Ollama や LM Studio がダメなのではなく、あなたのそのパソコンの「体格」が、どれだけ大きなモデルを動かせるかを決めているのです。

> **ポイント:** ローカルかクラウドかを選ぶには、まず自分にこう問いましょう：**「この用事で、最も大事なのはデータを外に出さないことか、それとも効果が最高であることか？」** データを外に出したくない・オフラインで使いたい・無課金にしたい――ローカルには代えがたい価値があります。最上位の効果がほしい・手軽がいい――おとなしくクラウドを使いましょう。これは第17章の「モデル/製品/API/エージェント」の地図と同じ道理です：まず需要がどの層にあるかを見極めてから、何を使うかを論じます。

## 三、ダウンロード前にひと目：あなたのパソコンは耐えられるか

これはローカルモデルで最も実際的で、しかも初心者が最も見落としやすい一つの関門です。**すべてのモデルをあなたのパソコンが動かせるわけではありません。**

たとえるなら：モデルは「荷物」、あなたのパソコンは「車」のようなものです。荷物には大小があり、車のトランクにも大小があります――**箱に収まらなければ、車は走り出せません**。モデルというこの「荷物」の大きさは、主に次の二つがそれを収められるかで決まります：

- **メモリ(RAM)**：パソコンが一時的に作業する空間。モデルは動くときに丸ごとメモリに「広げ」られ、モデルが大きいほど食うメモリも多くなります。
- **VRAM(ビデオメモリ)**：グラフィックボード(GPU)上の専用メモリ。多くのモデルはグラボで動かすとずっと速くなりますが、グラボの「専用空間」は通常メモリよりさらに逼迫しています。

実際的に言うとどういう意味か：ダウンロードできる各ローカルモデルには、配布元がたいてい **おおよそどれだけのメモリ/ビデオメモリが必要か** を記しています。ダウンロード前に、その要件をあなたのパソコンの構成と照らし合わせます――**収まってこそ、滑らかに動きます。構成を超える大きなモデルを無理に押し込めば、軽くてもひどく遅くなり(スライドショーのようにカクつく)、重ければそもそも動き出しません。**

```text
ダウンロード前の「ひと目合わせ」の動作：

  モデルが記す要件：      あなたのパソコン：    結論
  ─────────────         ─────────           ──────
  16 GB メモリが必要  vs  あなたは 8 GB    →  動かない / 極めて遅い
  8 GB メモリが必要   vs  あなたは 16 GB   →  問題なし
  （具体的な数値はモデル配布元/公式ドキュメントに従う、本書は書き切らない）
```

あるモデルがどれだけ大きく要るか、自分のパソコンがどれだけあるかをどう知るか？ これらの **具体的な数値と確認方法は各社で異なり、しかもよく更新されるので、公式ドキュメントに従ってください**。あなたはまずこの意識を持つだけでよいです：**ローカルモデルには「ハードウェアの敷居」があり、ダウンロード前にまず構成をひと目合わせる** こと。ダウンロードし終えてから動かないと気づいて、半日無駄に待つことのないように。

> **ポイント:** 「ローカルモデルを動かせるか」は、あなたが動かしたいかどうかではなく、あなたの **マシンが耐えられるか** で決まります。一つの習慣を身につけましょう：ローカルモデルを見たら、まずそれが記すメモリ/ビデオメモリの要件を探し、それから自分のパソコンの構成を見る――このステップは一分しかかからず、「数 GB をダウンロードしたのに、動かすとフリーズする」という骨折りを省けます。

## 四、量子化：大きなモデルを少し「やせさせて」載せやすくする

こう聞きたくなるかもしれません：では大きなモデルは普通のパソコンとは完全に縁がないのか、と。そうとも限りません。ここでよく耳にする言葉――**量子化（quantization）** があり、一言で分かる価値があります。

先の「荷物」のたとえを続けます：量子化は、荷物の中身を **真空圧縮袋で押し縮める** ようなもので、体積が小さくなり、小さなトランクに収めやすくなります。代償は――押しすぎると、服が少ししわになるかもしれない、ということです。

実際は何かに立ち返ると：量子化は **モデルをより小さく、よりメモリ節約に「圧縮」する** 技術で、本来動かせなかったモデルを、普通のパソコンでも動かせる可能性を与えます。代償は **押すほど通常、効果が少し落ちる**(回答の質がやや下がるかもしれない)ことです。だから同じモデルにいくつもの「圧縮の段階」が選べることをよく目にします――軽く押せば効果はよいがより構成を食い、重く押せば資源を節約するが質が少し割り引かれます。

> **ポイント:** 量子化はローカルモデルの世界でよくあるトレードオフです：**少しの「質」で「あなたのパソコンで動かせること」を買う** のです。普通の人はそれがどう実装されるかを分かる必要はなく、ただこう知っていればよいです――同じモデルにいくつもの版が選べるとき、それはたいてい異なる量子化の段階で、**「動かせる」と「うまく答える」の間でバランス点を一つ選ぶ** ものです。具体的にどう選ぶか、各段でどれだけ差があるかは、公式ドキュメントに従います。

## 五、Ollama と LM Studio：普通の人がモデルを動かすのを助けるツール

ここまで「ローカルモデル」を語ってきましたが、モデル自体はただのコアです(第17章を覚えていますか――モデルは「思考する頭」で、何かでそれを包んで初めて使いやすくなります)。**裸のモデルと直接付き合うのはとても骨が折れます**：ダウンロードし、設定し、動かし、しかもそれと対話する場所が要ります。

**Ollama と LM Studio は、これらの面倒事をあなたの代わりに一手に引き受けるツール** です――普通の人が比較的手軽にローカルでモデルを **ダウンロード** し、**実行** し、**それと対話** できるようにしてくれます。

ここでついでに一つよく出る言葉を補います：あなたがこうして自由にダウンロードし、自分のパソコンに入れて動かせるモデルは、たいてい **オープンウェイトモデル（open-weight model）** に属します――簡単に言えば、配布元がモデルを「公開」し、みんながローカルにダウンロードして使うことを許したものです。こうしたダウンロードできるモデルがあるからこそ、ローカルというこの道が成り立ちます。一方、最も最先端のクラウドモデルの多くはダウンロードを公開しておらず、その製品や API を通じてしか使えません(第17章)。どのモデルがオープンウェイトか、商用できるかは、各社の条項が異なるので、**公式ドキュメントに従ってください**。

- おおよその印象として(**具体的な機能・インターフェース・対応モデルは公式ドキュメントに従う**)：それらは通常、利用可能なローカルモデルの閲覧とダウンロードを助け、ダウンロードした複数のモデルを管理し、直接チャットできる入口を提供します。
- 両者は立ち位置が少し異なり、それぞれ重点がありますが、普通の人にとって目的は同じです：**「ローカルでモデルを動かす」という本来かなり技術的なことを、数回クリックすれば始められるようにする** こと。

具体的にどう入れ、どうモデルを落とし、コマンドやボタンがどんな見た目かは――これらは **バージョンによって頻繁に変わるので、本書は書き切らず、それぞれの公式ドキュメントに従ってください**。本当に手を動かす深い操作は、付録 C で改めて詳しく扱います。

> **ポイント:** Ollama / LM Studio と「モデル」自体を混同しないでください。それらは **モデルではなく**、モデルを **ダウンロードし、動かし、使えるように** してくれる「執事」です――レストランがシェフに対するように(第17章)。執事を変えても、動かすのは同じ一群のローカルモデルです。本当に効果を決めるのは、あなたが動かすのが **どのモデルか**、あなたの **パソコンがどれだけ強いか** であって、どの執事を使うかではありません。

## 六、よくある誤解、まとめて訂正

| 誤解 | 実際 |
| --- | --- |
| ローカルモデルは必ずクラウドより強い | むしろ逆――ローカルはあなたのマシンの計算力に制限され、同条件ではクラウドの最上位に **及ばないことが多い**。強弱は **具体的な型番＋あなたのハードウェア** により、「ローカル」の二文字では決まらない |
| ローカルモデル ＝ 絶対安全 | より **秘匿** である(データがローカルを出ない)のは確かだが、「秘匿」は「安全」とは違う――モデルは同じくハルシネーションし、間違える(第9章)。素性の知れないモデルをダウンロードするのもリスクがある |
| Ollama を入れれば「より良い AI」が手に入る | Ollama/LM Studio は **実行ツール** にすぎず、モデル本体ではない。本当に働くのはあなたがダウンロードしたそのモデル |
| 適当にモデルを一つ落とせば動く | まずそのモデルの **メモリ/ビデオメモリ要件** があなたのパソコンと合うかを見る。さもなければ極めて遅く、あるいは動き出さない |
| ローカルはギーク向けで、自分には関係ない | **データを外に出さないことを特に気にする、オフラインで使いたい、使用量が多い** なら、ローカルは現実的な価値のある一つの道――ただし少し設定の敷居がある |

## まとめ

- **ローカルモデル** を選ぶ人が狙うのは三つ：**秘匿**(データがローカルを出ない)、**オフライン**(ネットがなくても使える)、**回数で課金されない**――どれも「モデルが自分のマシンで動く」ことに由来する。
- ローカル vs クラウドは **トレードオフの問題**：クラウドは **強力・手軽だがデータが外に出る・従量課金**。ローカルは **秘匿・オフライン・無課金だが、マシンに制限され、敷居があり、通常クラウドほど強くない**。
- ダウンロード前に **まずハードウェア要件を見る**：モデルはあなたの **メモリ/ビデオメモリ** に「広がる」必要があり、収まらなければ動かない、あるいは動き出さない(具体的な数値は公式ドキュメントに従う)。
- **量子化** は、少しの質で「普通のパソコンで動かせること」を買うよくあるトレードオフ。同じモデルの複数の版は、たいてい異なる量子化の段階。
- **Ollama / LM Studio** は、普通の人がローカルでダウンロード・実行・対話するのを助ける **ツール** であって、モデル本体ではない。具体的な使い方は公式ドキュメントに従い、深い操作は付録 C 参照。

次章では視点を変えます：モデルがいくら強くても、「言う」だけでは足りません――どうやってそれに **外部のツールとデータ** をつなぎ、あなたが指定した資料を読ませ、あなたの決まった流れに沿って用事をさせるか？ それには Skills と MCP を語ることになります。

## 小テスト

> 全 6 問。概念・誤解・場面・操作の 4 種類をカバーしています。答えと解説は各問の下の引用ブロックにあります――まず自分で考えてから、照らし合わせてください。

1. **[基礎・概念問題]** 人々が自分のパソコンでローカルモデルを動かしたいと思う、主な狙いは次のどの三つか？
   - A. 秘匿、オフライン、回数で課金されない
   - B. 必ずより強い、必ずより速い、必ずパソコンが安く買える
   - C. 自動でネット接続、自動更新、自動バックアップ
   - D. 永遠に間違えない、永遠にハルシネーションしない
   > **答え A。** ローカルモデルの三つの甘い汁――データがローカルを出ない(秘匿)、ネットがなくても使える(オフライン)、入れた後は課金されない――は、どれも「モデルが自分のマシンで動く」というこの一事から来ます。B はローカルを「より強い・より速い」と取り違え(実際はクラウドの最上位に及ばないことが多い)、D は「秘匿」を「間違えない」と誤解しています。モデルは同じくハルシネーションします(第9章)。

2. **[基礎・概念問題]** 「ローカル」と「クラウド」について、最も正確なのはどれ？
   - A. ローカルは永遠にクラウドより良い
   - B. クラウドは強力・手軽だがデータが外に出る・従量課金。ローカルは秘匿・オフライン・無課金だがマシンに制限され、通常クラウドほど強くない
   - C. クラウドはネット接続が不要で、ローカルはネット接続が必要
   - D. 両者は完全に同じで、名前が違うだけ
   > **答え B。** これはトレードオフの問題で、「どちらが永遠に良い」はありません。A は最もよくある誤解です。C は両者のネット接続の関係を **逆に言って** います(クラウドこそネット接続が要り、ローカルは不要)。D はデータの行き先、課金、能力など一連の実打実の違いを無視しています。

3. **[基礎・誤解問題]** 「ローカルモデルは必ずクラウドのものより強く、より賢い。」――どこが間違い？
   - A. 間違いではない、ローカルこそより強い
   - B. ローカルはあなたのパソコンの計算力に制限され、同条件ではクラウドの最上位に及ばないことが多い。強弱は具体的な型番とあなたのハードウェアにより、「ローカル」の二文字では決まらない
   - C. ローカルは実はより弱く、永遠に使えないのが間違い
   - D. ローカルとクラウドは完全に異なるものを動かしているのが間違い
   > **答え B。** クラウドの背後は普通の人には買えない巨大な計算力で、あなたのパソコンで動かせるのはたいてい「スリム版」です。強弱を決めるのは **どのモデル＋あなたのハードウェア** であり、「ローカル」の二文字自体は強さを意味しません。この誤りを犯す人は、たいてい「自分のパソコンに入れる」を「より高級」と読み違えています。C はもう一方の極端に走っています(ローカルにも代えがたい価値があります)。

4. **[応用・誤解問題]** 「ローカルモデルはデータが外に出ないから、絶対に安全で、間違えない。」――どこが正しくない？
   - A. 完全に正しい
   - B. 「より秘匿」≠「絶対安全」：モデルは同じくハルシネーションし、間違える。素性の知れないモデルをダウンロードすること自体にもリスクがある
   - C. ローカルモデルは実はこっそりデータをアップロードする
   - D. ローカルモデルにはいかなる秘匿の利点もない
   > **答え B。** 「データがローカルを出ない」ことは確かにそれをより **秘匿** にしますが、「秘匿」と「安全/正確」は別のことです――その回答はなお間違いうる(第9章のハルシネーションは同じく存在する)し、信頼できない出所からモデルを落とすこと自体にリスクがあります。「秘匿」を「安全」と同一視するのが、この問いが正したい混同です。D はその実在する秘匿の利点を否定しており、同じく正しくありません。

5. **[基礎・場面問題]** あるローカルモデルが気に入りました。ダウンロードする前に **まず最もすべき** 一つのことは？
   - A. そのままダウンロードを押し、動かなかったらそのとき考える
   - B. それが記すメモリ/ビデオメモリの要件をひと目見て、自分のパソコンの構成が耐えられるか照らし合わせる
   - C. まずクラウドの会員を解約する
   - D. パソコンの他のソフトを全部アンインストールする
   > **答え B。** モデルはあなたのメモリ/ビデオメモリに丸ごと「広がる」必要があり、**収まらなければ極めて遅い、あるいは動き出しません**。一分かけて構成をひと目合わせれば、「数 GB を落としたのにスライドショーになる」骨折りを省けます(具体的な数値は公式ドキュメントに従う)。A は最も踏みやすい落とし穴です。C・D はどちらも「ハードウェアが足りるか」という本当の関門に的を当てていません。

6. **[基礎・操作/観察問題]** まず自分のパソコンのメモリ(RAM)がおよそ何 GB かを調べます。それから任意のローカルモデルツール(Ollama / LM Studio など、**具体は公式ドキュメントに従う**)を見て、同じモデルにしばしば大きさの異なる複数の版があることを確かめます。観察してみてください：それらより小さな版は、より資源を節約するが、効果は少し劣るのではないでしょうか。
   > **観察できるはず：** 同じモデルの複数の版は、たいてい異なる **量子化（quantization）** の段階です――押すほど・ファイルが小さいほど・メモリを節約するが、回答の質はやや下がるかもしれません。軽く押すほど・構成を食うほど・効果はよくなります。これこそローカルモデルの核心のトレードオフです：**「動かせる」と「うまく答える」の間でバランス点を選ぶ** こと。一度自分のメモリとモデルの大きさを合わせてみれば、「自分のこのパソコンはどれだけ大きなモデルを動かせるか」という直感が身につきます(各段でどれだけ差があるかは公式ドキュメントに従う)。
