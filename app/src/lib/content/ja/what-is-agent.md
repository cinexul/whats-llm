ここまでであなたが知っている AI は、本質的に「**質問に答えてくれる相手**」です。あなたが一言聞けば、一言答える。提案をくれ、下書きをくれ、考え方をくれます――けれど **実際に手を動かすのは、いつもあなた** です。AI が「このファイルを直したほうがいい」と言っても、ファイルはやはりあなた自身が直さなければなりません。

本章で扱う **エージェント（Agent）** は、**質的に違うもの** です。「どうやればいいか教える」だけではなく、**自分で手を動かして実行します** ――自分でファイルを読み、自分でコマンドを走らせ、自分でネットにつなぎ、自分で他のプログラムを呼び出して、一つの目標に向かって一歩ずつ前に進めていきます。第17章の「四層の地図」の一番外側がこれです。本章でこれを徹底的に説明し、第4部で学ぶ「コーディングエージェント」(第23・24章)の土台も固めます。

一言だけ覚えておいてください。本章はすべてこれを説明しています――**エージェントのすべての能力とすべてのリスクは、同じ一つのことから来る。「本当に手を動かせる」ことです。**

## 一、「質問に答える AI」から「手を動かす AI」へ

まず、この二種類の AI の境界線をはっきり見ましょう。

| | チャット式 AI(あなたが慣れているもの) | エージェント(本章) |
| --- | --- | --- |
| 何をくれるか | ひとまとまりの **回答**(提案・下書き・考え方) | 一つの **結果**(物事が本当に行われた) |
| 誰が実行するか | **あなた** が答えに従い、自分で手を動かす | **エージェント自身** が実行し、あなたは傍らで見守る |
| どう進めるか | こちらが一言聞き、一言答える | 目標に向かって、自分で何歩も前に進む |
| たとえるなら | **コンサルタント**：案を出すだけ | **アシスタント**：直接あなたの代わりに用事を済ませる |

前章のシェフのたとえを続けましょう(第17章)。チャット式 AI は「レシピを口述する」だけのシェフ――どう作るか教えてくれますが、鍋を握るのはやはりあなたです。エージェントは **本当に厨房に立つ** シェフ――「今夜は家庭料理を」と言えば、自分で冷蔵庫を見て、献立を立て、足りない材料を補い、一品ずつ炒め、最後にコンロまわりまで片づけます。

> **ポイント:** エージェントを「より賢いチャットボット」と理解しないでください。決定的な違いは **「より賢い」ことではなく「手を動かす」こと** にあります。一方は答えるだけ、もう一方はあなたのものを本当に操作します(ファイルを直す、コマンドを走らせる、ネットで注文する……)。これは能力の階層が跳躍したのであって、同じものが少し強くなっただけではありません。

では、その「手を動かす」能力はどこから来るのでしょうか。答えは前部で伏線を張っておいたもの、**ツール** です。

## 二、肝心な部品：ツール呼び出し(tool use)

モデル自身は一つのことしかできません：**前の文脈にもとづいて文字を生成する**(第4章)。手も目もなく、あなたのファイルに触れることも、ネットにつなぐこともできません。ではエージェントはどうやって「手を動かす」のでしょうか。

モデルに一式の **ツール(tool)** を備えさせ、**ツール呼び出し(tool use)** を覚えさせることによります。つまり、モデルは文字を生成する過程で、**あるツールを使うよう「要求」** できます(「このファイルを読んで」「これを検索して」「このコマンドを走らせて」)。外側のプログラムが代わりに本当に実行し、その **実行結果** をモデルに戻して、続きを考えさせるのです。

```text
ツールのないモデル：
   あなたが聞く → モデルは「頭の中のもの」だけで答える → ひとまとまりの文字（現実世界に触れられない）

ツールを備えたエージェント：
   目標 → モデルが「まずこのファイルを読む必要がある」と言う → プログラムが本当に読む → 内容を戻す
        → モデルが読んで「次にこれを検索」と言う → プログラムが本当に検索 → 結果を戻す
        → ……こうして往復し、目標が達成されたと判断するまで続く
```

- ツールはさまざまです：ファイルの読み書き、コマンドの実行、ネット検索、データベース照会、外部サービスの呼び出し……(あるエージェントがどのツールを備え、どう設定するかは、**公式ドキュメントを確認してください**。AI にツールをつなぐことは、第22章と第6部でも改めて専門に扱います。)
- ここでの分業に注意してください：**「どのツールを使うか、何のために使うか」を決めるのはモデルで、「本当に実行する」のは外側のプログラム** です。モデルが頭を働かせて指示を出し、プログラムが手を動かす――この二つが合わさって、初めて「手を動かす AI」になります。

> **ポイント:** 「エージェント＝モデル＋ツール＋一つの目標」。モデルは相変わらず文字を生成するだけのコアですが、**生成した文字が「何かをする」指示に変わり、本当に実行されるとき**、「言うだけ」から「やる」へと変わります。ツール呼び出しこそ、モデルと現実世界の間につながれた「手」なのです。

## 三、エージェントの定番の動き：「調査→実行→確認」のループを自分で回す

一度ツールを呼べるだけでは、まだエージェントの真髄ではありません。本当に「仕事ができる」ように見せているのは、**自分でループを回せること** です。多段で進め、あなたが一歩ずつ指示しなくてよいのです。

このループはおおよそこうなります：

```text
        ┌─────────────────────────────────────────┐
        │  ① 調査：現状を見て、必要な情報を集める     │
        │  ② 実行：分かったことをもとに、一歩操作する │
        │  ③ 確認：結果が正しいか、目標に近づいたか   │
        │  ④ 調整：違えば方法を変え、また ① に戻る    │
        └───────────────┬─────────────────────────┘
                        │ 「目標達成」と判断するまで
                        ▼
                     あなたに引き渡して最終チェック
```

プログラミングを含まない例で感覚をつかみましょう。あるエージェントに「このフォルダの画像を全部、日付ごとに整理して」と頼んだとします。動きはこうなるかもしれません。まずフォルダの中身を **見て**(調査)→ 各画像の情報を **読み**、分類フォルダを **作り**、ファイルを **移動し**(実行)→ 抜けや置き違いがないか **照合し**(確認)→ ある一枚に日付情報がないと気づいて、**別のやり方** で処理する(調整)……こうして一巡ずつ、整理し終えるまで続けます。この過程全体を **エージェント自身が進め**、あなたは肝心なところだけ見守ります。

これはまさに第17章の一文の展開です。**モデルにツールを備えさせ、目標を中心に「調査→実行→確認→また調整」のループを自分で回させる。** この「自分でループを回す」能力こそ、チャットボットと一線を画す核心です。

> **ポイント:** チャット式 AI は「**あなたが一押しすれば、一歩進む**」。エージェントは「**目標を与えれば、自分で一区間進む**」。このループを自律的に回せるということは、「何歩も必要で、途中で結果に応じて臨機応変に対応する」タスクをこなせるということです。これが本当に役立つところであり、同時に次節で述べる、本当に危険なところでもあります。

## 四、能力とリスクは、同じ一枚のコイン

これは本章で最も心に刻むべき点です。**エージェントは強いところが、そのまま危険なところ。両者は同じ一つのことだからです。**

チャット式 AI は、たとえ間違えても害は限られます。間違ったことを **言った** だけで、従うかどうか、どうするかは **すべてあなたの管理下** にあります。けれどエージェントは違います。**あなたのものを本当に操作します**。だから：

| この能力 | **役に立つ** 面 | 同時に **リスクになる** 面 |
| --- | --- | --- |
| あなたのファイルを手を動かして直せる | 本当に用事を片づけてくれる | 動かしたくないものを **直し間違え・削除し間違える** かもしれない |
| 自分でコマンドを走らせられる | 面倒な手順を自動化する | **予想外の結果** を引き起こすかもしれない |
| 次の一手を自分で決められる | 一歩ずつ見張らなくてよい | その判断が **誤る** かもしれず、しかも自分で進み続ける |
| ネットにつなぎ、外部サービスを呼べる | リアルタイム情報と外部能力を使える | **望まない対外的な動作** を引き起こすかもしれない |

つまるところ、**「あなたの代わりに用事をこなせる」ことと「あなたの代わりに事故を起こせる」ことは、同じ能力の裏表** です。あなたは「手」を与えました。エージェントはこの手で助けることも、この手で問題を起こすこともできます。しかも問題を起こすとき、たいてい **立ち止まってあなたに聞かず**、自分の判断のまま進み続けます。

> **ポイント:** 第17章の一文がここで地に着きます。**エージェントの能力とリスクは一体** です。だからエージェントは決して「放任」で使ってはならず、二つのものを備えなければなりません。**①許可/確認の仕組み**(肝心な動作はまずあなたに確認し、あなたが承認してから行う)、**②巻き戻せる環境**(万一間違えても、元に戻せる)。この二つは面倒を増やすためではなく、**「手を動かす AI」を安全に使うための前提** です。具体的にどう許可を設けるか、どう巻き戻すかは、第4部と第6部で専門に扱います(第26・27・40章)。本書でも繰り返し強調します。**手元のものに手を動かすときは、必ず Git で始め、まずブランチを切る**(第24章)。いつでも戻れるようにです。

これは、なぜ本書が「安全な使い方」をとても重い位置に置くのかも、ついでに説明してくれます。エージェントの「手を動かす」恩恵を享受したいほど、その「手を動かす」ことを制御可能な範囲に収めなければならないのです。

## 五、第17章への回収：エージェントは「四層の地図」のどこにいるか

本章を前部の地図につなぎ直せば(第17章)、ひと目でわかります。

```text
   ┌──────────────────────── エージェント（自分で流れを回す）← 本章
   │   ┌──────────────────── 製品（人が使うインターフェース）
   │   │   ┌──────────────── API（プログラムが使うインターフェース）
   │   │   │   ┌──────────── モデル（思考するコア）
あなた→目標   注文   呼び出し       次の一段の文字を生成
```

- 一番内側は常に、**文字を生成するだけ** のモデル(第4章)。
- エージェントは、その外側に **ツールを備え、目標を加え、自分でループを回させて** できた一番外側の層です。
- そしてエージェント内部の「頭を働かせる」部分は、しばしば **API**(第17章)を通じてモデルを呼び出しています。つまりこれらの層は排他的ではなく、幾重にも包み込んでいるのです。

> **ポイント:** ある AI に出会って「これはエージェントか」を判断したいとき、一つ問えば十分です。**「言う」だけか、それとも「本当に手を動かす」(ファイルを直す、コマンドを走らせる、ネットにつなぐ、ツールを呼ぶ)か?** 言うだけなら、まだ製品/モデルの層。手を動かし、しかも目標を中心に自分で前に進めるなら、エージェントの性質を帯びています。これこそ第17章が教えた「自分が向き合っているのはどの層か」の判断法です。

## 六、よくある誤解、まとめて訂正

| 誤解 | 実際 |
| --- | --- |
| エージェントは「より賢いチャットボット」だ | 決定的な違いは「より賢い」ことではなく **手を動かす** こと。チャット AI は言うだけ、エージェントはあなたのものを本当に操作し、自分でループを回す |
| エージェントは何もないところから「手を動かせる」 | 支えているのは **ツール呼び出し(tool use)**。モデルが「あるツールを使う」要求を出し、外側のプログラムが本当に実行し、結果を戻す――モデルが頭を、プログラムが手を |
| 目標を与えたら、放任して自分でやらせればよい | エージェントは本当に操作し、しかも立ち止まって聞かないことがある。**許可/確認＋巻き戻せる環境** を備えて初めて安全に使える(第26・27・40章) |
| 能力が強いから、より安心してよい | 能力とリスクは **同じ一枚のコイン**。用事をこなせる＝事故を起こせる。できるほど「手を動かす」ことを管理しなければならない |
| エージェントとモデル/製品/API は四択一だ | これらは **幾重にも包み込む**。エージェント＝モデル＋ツール＋目標、内部はしばしば API でモデルを呼ぶ(第17章) |
| ツールはモデル自身が実行する | 「どのツールを使うか」はモデルが決め、「本当に実行する」のは外側のプログラムが行う。分業が異なる |

## まとめ

- **エージェント（Agent）** は「**本当に手を動かす**」AI：どうやるか教えるだけでなく、自分でファイルを読み、コマンドを走らせ、ネットにつなぎ、プログラムを呼んで、目標を中心に用事をやり遂げる。
- 手を動かす腕前は **ツール呼び出し(tool use)** から来る。モデルは文字を生成するだけだが、生成した文字が「あるツールを使う」要求に変わり、外側のプログラムに本当に実行されるとき、「言う」から「やる」へと変わる。
- 定番の動きは **「調査→実行→確認→調整」のループを自分で回すこと**。目標を与えれば、自分で一区間進む。あなたが一歩ずつ指示するのではない(第17章の四層に呼応)。
- **能力とリスクは同じ一枚のコイン**。用事をこなせれば、事故も起こせる。間違えるとき、しばしば立ち止まって聞かない。だから **許可/確認の仕組み＋巻き戻せる環境** を備えなければならない(第26・27・40章。手元で手を動かすときは必ず Git で始める、第24章)。
- 第17章の地図上で、エージェントは **一番外側**。モデル＋ツール＋目標、内部はしばしば API でモデルを呼ぶ。「エージェントか」を判断するには、**本当に手を動かすか** を見る。

次章では方向を変えて、もう一つの「AI に外付けする」よくあるやり方を見ます。**RAG** です。どうやって AI に資料のデータベースを外付けし、答える前にまず「本を引かせる」かを扱います。

## 小テスト

> 全 6 問。概念・誤解・場面・操作の 4 種類をカバーしています。答えと解説は各問の下の引用ブロックにあります――まず自分で考えてから、照らし合わせてください。

1. **[基礎・概念問題]** ある AI が「チャット式 AI」か「エージェント（Agent）」かを分ける、最も肝心な境界線は？
   - A. どちらが使うモデルのパラメータが多いか
   - B. 「言う」(回答をくれる)だけか、それとも「本当に手を動かす」(ファイルを直す、コマンドを走らせる、ネットにつなぐ、ツールを呼ぶ)うえに、目標を中心に自分で前に進めるか
   - C. どちらが速く答えるか
   - D. どちらのインターフェースが見栄えするか
   > **答え B。** エージェントの要は「より賢い」ことではなく「手を動かす」こと。あなたのものを本当に操作でき、しかも自分で多段のループを回せます。A・C・D はどれも無関係か副次的な要素です。「パラメータが多い/より賢い」をエージェントの定義とするのは、まさに最もよくある誤解です。

2. **[基礎・概念問題]** エージェントが現実のファイルやネットを「手を動かして」操作できるのは、どんな仕組みによるか？
   - A. モデルが自分で手を生やした
   - B. **ツール呼び出し(tool use)**：モデルが「あるツールを使う」要求を出し、外側のプログラムが本当に実行し、結果をモデルに戻す
   - C. モデルが自分をあなたのパソコンにダウンロードした
   - D. もっぱらモデルの記憶の中のものに頼る
   > **答え B。** モデル自身は文字を生成するだけ(第4章)で、「ツール呼び出しを要求 → プログラムが本当に実行 → 結果を戻す」という仕組みを通じて間接的に「手を動かし」ます。**モデルが頭を、プログラムが手を**。A・C は「手を動かす」を文字どおりに受け取った誤解です。D は誤りで、手を動かすのは、まさにモデルの記憶 **以外** の現実の情報と操作を使うためです。

3. **[基礎・概念問題]** エージェントの代名詞となる働き方の、最も的確な記述は？
   - A. こちらが一言聞き、一言答え、聞き終われば止まる
   - B. 目標を与えれば、自分で「調査→実行→確認→調整」のループを回し、多段で進めて完了まで至る
   - C. ありうる答えをすべて一度に列挙する
   - D. 案を出すだけで、やるのはやはりあなた
   > **答え B。** 「自分でループを回し、多段で進める」ことこそ、エージェントがチャットボットと一線を画す核心です。A・D が描いているのは、まさに **チャット式 AI**(あなたが一押しすれば一歩進む、案を出すだけ)です。C はエージェントの働き方とは無関係です。

4. **[応用・誤解問題]** 「エージェントは能力が強く、自分で用事を片づけられるから、安心して丸投げして放任してよい。」どこが間違い？
   - A. 間違いではない。強いほど手放してよい
   - B. 能力とリスクは **同じ一枚のコイン**：用事をこなせれば事故も起こせる(直し間違え、予想外の結果を出す)。しかも間違えるとき、しばしば立ち止まって聞かない――許可/確認と巻き戻せる環境を備えなければならない
   - C. エージェントは間違えない
   - D. 間違えたら自動で取り消してくれる
   > **答え B。** これは本章で最も心に刻むべき点です。エージェントの「手を動かす」は、その価値であると同時にリスクの源です。あなたが予想しないことを勝手に行い、しかも自分で進み続けうるからこそ、**許可の仕組み＋巻き戻せる環境** で最後の備えとする必要があります(第26・27・40章。手元で手を動かすときは Git で始める、第24章)。A は危険です。C・D はどちらもエージェントを当てにしすぎています。本当に間違えるし、自動で後始末もしてくれません。

5. **[基礎・場面問題]** あるエージェントに、パソコン上の大事なフォルダの整理(移動・改名・重複削除)を頼みます。手を動かす前に、まず最もすべきことは？
   - A. そのまま走らせ、間違えないと信じる
   - B. まずフォルダを **バックアップ** する(または巻き戻せる環境で操作する)、そして「削除・上書き」のような肝心な動作は **まずあなたに聞いてから** 行うよう求める
   - C. パソコンの電源を切る
   - D. 一度に全部やらせて、最後に結果を見る
   > **答え B。** エージェントは本当に手を動かすので、**直し間違え・削除し間違えは、そのまま本当の直し間違え・削除し間違え** になります。だから「巻き戻せる(バックアップ/Git)＋肝心な動作はまず確認」が、エージェントを使う基本の安全姿勢です(第26・27・40章)。A・D はどちらも「放任」であり、まさに本章が繰り返し警告する危険な使い方です。C は何の問題も解決しません。

6. **[応用・操作/観察問題]** あなたが触れられる、「手を動かす」能力をもった AI ツール(ネット検索ができる、アップロードしたファイルを読める、ツールを呼べる類)を一つ見つけます。そのツールに **何歩も必要な** 小さなタスクを与え、「自分でまず調べ、それから手を動かし、結果に応じて調整する」かを観察しましょう。さらに **チャットしかできない** AI で同じことをして、両者の違いを体感してください。
   > **観察できるはず:** ツールを備えたほうは、「調査→実行→確認→調整」のループを示します。自分で調べ、実行し、途中の結果に応じてやり方を変え、**何歩も前に進み** ます。チャットしかできないほうは、「こうすべき」というひとまとまりの文字をくれるだけで、本当に実行するのはやはりあなたです。一度自分の手で比べれば、本章の核心に触れられます。エージェントの違いは **より賢いことではなく、本当に手を動かすこと** にあります(忘れずに。手を動かす＝エージェントを管理する。肝心な動作はまずあなたに確認させる)。
