詩を書き、コードを直し、長い文章を要点へきれいにまとめてくれます。けれど、興ざめなことに二つ、きっと出くわしているはずです。一つは**もっともらしく堂々と間違える**こと――まったく存在しない論文を引き、いるはずのない「専門家」の名を挙げ、数字をいかにもそれらしくでっち上げます。もう一つは**こちらを覚えていない**こと――昨日伝えたばかりのことなのに、今日新しい会話を開くと、きょとんとしています。

この二つは、AIが「頼りない」とか「サボっている」からではなく、その働き方がもたらす必然の結果です。この章では、両方を一度で説明しきります。**ハルシネーション**（なぜ間違えるのか）、そして**知識 vs 記憶**（何を覚えていて、何をすぐ忘れるのか）。この章が分かれば、AIを「どこまで信じ、ものをどこに置くべきか」の見当がつきます――本書の中でも最も「落とし穴を踏まない」ことに関わる章です。

## 一、ハルシネーション：存在しないものを、やけに自信満々に語る

**ハルシネーション（hallucination）** とは、モデルが**存在しない、あるいは誤った**内容を、**いかにもそれらしく、自信たっぷりに**語ることを指します。肝心なのは「間違えた」ことではありません――誰でも間違えます――そうではなく、**まったく破綻なく、しかもやけに自信満々に間違える**ことにあります。

なぜこうなるのでしょうか。答えは第4章の一文の中にあります。

> **モデルが追い求めるのは「本物らしい」ことであって、「本物である」ことではありません。** モデルの本分は、前の文をもとに「続けて最もそれらしく、最も滑らかな語」を選ぶこと（第4章）で、真偽を照合できる「事実データベース」を持っているわけではありません。ですから「知らない」とき、立ち止まって「自信がありません」とは言わず、言語の惰性に沿って、**読んで最も筋の通る答えを補ってしまいます**。偽の論文が本物らしく見えるのは（それらしいタイトル、それらしい著者、それらしい学術誌）、何千何万もの本物の論文がどんな形かを見てきて、その形をまねて一本こしらえたからです。嘘をついているのではなく（嘘をつくにはまず真実を知っていなければなりません）、「**話を滑らかに続けて**」いるだけで、続けるうちに偽物が出てきてしまうのです。

成り立ちを理解すれば、対策は自然と出てきます――三つの手、弱いものから強いものへ。

| 対策 | やり方 | 何に効くか |
| --- | --- | --- |
| **自分で再確認する** | 重要な事実（数字、引用、人名、日付、法律・医療の結論）は一律に自分でもう一度調べる | 最後の砦、最も重要な一手 |
| **根拠を出させる** | 「出典／ソースを列挙して」「なぜそう言えるのか説明して」と要求する | 根拠がない、または根拠が怪しければ、それが警報 |
| **材料を与える** | 信頼できる資料を貼り付けて、「**これだけ**に基づいて答えて」と言う | 出どころからでっち上げの余地を抑える |

> **ポイント:** 最も直感に反するけれど、最も頭に刻むべき判断――**滑らかで自信満々に語るほど信頼できる、というわけではありません**。滑らかで自信満々なのは「滑らかに続けている」ことを示すだけで、「本物かどうか」とはまったく関係ありません。本当に安全な習慣は逆です。**重要なこと、AIが口をついて出した事実ほど、自分の手で検証する。** 三つ目の手「材料を与えて、材料に基づいて答えさせる」こそ、のちの RAG（第20章）という技術一式の核心となる考え方です――正しく覚えていることに賭けるより、正しい資料をその場で与えるほうがよいのです。

## 二、知識 vs 記憶：一枚の表で「覚えている」と「すぐ忘れる」を分ける

AIが内部に持っている情報は、実は**二種類**あり、性質がまったく違います。この二つを混同するのが、初心者の最もつまずきやすいところです。

- **知識（knowledge）**：学習時に**パラメータに焼き付けられた**常識と事実。今回与えた材料に依存せず口をついて出ます――「水の沸点」「江戸時代は明治時代の前か後か」と尋ねれば、直接答えられます。
- **記憶（memory）**：**今回の会話**の中だけに生きる一時的な情報で、すべてコンテキストウィンドウ（第3章の「机」）に支えられています。今回、自分の名前や好みを伝えた――これらは「記憶」で、**会話を閉じる、あるいはウィンドウから押し出されれば、消えます**。

並べて見ます。

| | 知識（knowledge） | 記憶（memory） |
| --- | --- | --- |
| どこにある | **パラメータに焼き付けられている**（学習時に学んだ） | **今回の会話のコンテキスト**の中だけ |
| どこから来た | 膨大な学習テキスト | 今回の会話で与えたもの |
| どれだけ持つ | 長期的にある（ただし「締め切り」がある、下記） | 閉じる／ウィンドウを超えると忘れる |
| 口をついて出るか | はい、材料を与えなくてよい | いいえ、すべて今回机に広げたものしだい |
| 典型例 | 「パリはフランスの首都」 | 「さっきピーナッツアレルギーだと言いました」 |

知識には避けて通れない硬い制約があります――**知識のカットオフ（knowledge cutoff）**。モデルの学習データには「最終期限」があり、**この時点より後**に起きたこと、出た新バージョン、現れた新語は、モデル**自身は知りません**。無理に尋ねれば、**ハルシネーション**を一つでっち上げてくる可能性が高いです。

> **ポイント:** 各モデルの知識のカットオフ時点は異なり、バージョン更新で変わります。本書では固定で書きません。**公式ドキュメントを参照してください**。覚えておくべき直感はこれだけです――**「最近何があったか」を尋ねるなら、口をついて出てきた答えは基本的に信じない**こと。インターネットで調べさせるか（第19章、調べた結果も再確認する）、最新の資料を貼り付けて読ませます。AIの「知識」はある瞬間のスナップショットで、自動では更新されません。

## 三、最も解いておくべき誤解：AIは人のように長く自分を覚えているのか？

ここが本章――そして本書――で最も大切で、最もはっきりさせておくべき点です。第3章の一文を、さらに深く掘り下げます。

長く話していると、つい「自分のことを知ってくれた」気がしてきます。けれど、第3章の結論を思い出してください。**モデル本体には、会話をまたいだ記憶がありません**。新しい会話を開けば、机は**まっさら**で、こちらについて何も知りません。同じ会話でも長く話しすぎれば、最初のほうの内容はウィンドウから押し出されて「忘れられて」しまいます。これは「物覚えが悪い」のではなく、**本体にそもそも会話をまたいでものを保存できる場所がない**のです。

けれど、きっとすぐにこう聞き返したくなるはずです。**では Claude の Projects、ChatGPT の Memory（メモリ）はいったい何なのか？ 自分の好みを「覚えて」いるのを、プロジェクト全体を覚えているのを、確かに見たぞ！** これははっきりさせるべき肝心な点です――

> **モデル本体には確かに会話をまたいだ記憶はありません。けれど、多くの製品はモデルの外側の「製品レイヤー」で「記憶」の能力を別途提供します**：たとえば長期記憶、プロジェクト記憶、プロジェクト文脈ファイル、アカウント単位の好み、企業のナレッジベースなどです（身近なところでは、Claude Projects、ChatGPT の Memory、IDE のプロジェクト文脈ファイル（CLAUDE.md など）、アカウント内の個人設定、社内ナレッジベースがあります）。ただしこれは**モデル本体に記憶が生えたのではなく、製品レイヤーが毎回、関連情報を自動で今回の会話のコンテキストに入れ直しているだけ**です。ですから、これらの機能を目にしても矛盾しません――カバーを開ければ、その下はやはり「記憶のないモデル＋毎回コンテキストを入れ直す」（まさに第3章の「机」）です。各社が具体的にどう覚え、どれだけ覚え、デフォルトでオンかオフか、削除できるかは、**公式ドキュメントを参照してください**。

この二つの層を分けて理解することは、どの製品の使い方を覚えるよりも価値があります。

| | モデル本体 | 製品レイヤーの「記憶」 |
| --- | --- | --- |
| 何か | 確率で続きを書くだけの中核 | 製品がモデルの外側に足した一層の機能 |
| あなたを覚えているか | **覚えていない**、毎回まっさらから始まる | 「覚えるべきもの」を保存してくれる |
| どうやって実現しているか | —— | **会話のたびに、これらの情報を自動で今回のコンテキストに入れ直す**（またあの机に戻る） |
| 「覚えている」範囲 | —— | 永遠に**製品が机に入れ直すと決めたその部分だけ**。入れ直されなかったものは、やはり知らない |

> **ポイント:** 「**底は記憶なし、上は入れ直しに頼る**」と理解すれば、これらの機能の挙動を予測できます。AIが「覚えて」いられるのは、永遠に製品が代わりにコンテキストへ入れ直したその部分だけ。覚えているはずだと思っても、製品が入れ直さなかったものは知りません。これは、設定したはずの好みをAIがときどき「わけもなく忘れる」理由も説明します――多くの場合、その情報が今回机に戻されなかったのです。ですからこれは不具合ではなく、「製品レイヤーの記憶」の働き方なのです。

## 四、前述の「誤解」をまとめて締める

本章には呼応し合う二つの誤解があり、並べて置くと最もはっきりします。

- **「自信満々に語るほど信頼できる」** ―― 間違い。自信は「滑らかに続けている」ことを示すだけで、真偽とは無関係。重要なこと、口をついて出したことほど、自分で検証する。
- **「AIは人のように長く私を覚えている」** ―― 間違い。モデル本体には会話をまたいだ記憶はない。目にする「覚えている」は、製品レイヤーが毎回情報をコンテキストに入れ直しているのであって、AI本人が覚えたのではない。

この二つは実は**同じ根底の事実の両面**です。AIには「真実データベース」もなければ「長期記憶の倉庫」もなく、あるのは学習時にパラメータへ圧縮された知識（締め切りあり）＋今回机に広げたコンテキストだけです。これを頭に刻めば、AIが「間違える」ことも「覚えていない」ことも、どちらももう不思議ではなくなります。

## 五、よくある誤解、まとめて正す

| 誤解 | 実際 |
| --- | --- |
| 滑らかで自信満々に語るほど、信頼できる | 滑らかで自信満々なのは「滑らかに続けている」ことを示すだけで、真偽とは無関係。重要な事実ほど自分で再確認する |
| 「間違える」のはたまの不注意で、注意すれば直る | ハルシネーションは「本物らしい」を追い、「本物である」を追わないことに由来する本質的特性。再確認・根拠を出させる・材料を与えることで防ぐ |
| AIは最新に起きたことを知っている | 知識には**締め切り**があり、その後のことはモデル本体は知らない。無理に尋ねればハルシネーションする。インターネットで調べるか資料を読み込ませる（公式ドキュメントを参照） |
| 今回伝えたことを、AIはずっと覚えている | それは「記憶」で、今回の会話の中だけに生きる。閉じる、または押し出されれば忘れる |
| 「記憶」機能があるのは、モデル本体が私を覚えた証拠 | **製品レイヤー**が毎回情報をコンテキストに入れ直しているのであって、モデル本体が覚えたのではない。具体的には公式ドキュメントを参照 |

## まとめ

- **ハルシネーション（hallucination）**：存在しない／誤ったものを、やけに自信満々に語ること。成り立ちは、「**本物らしい**」を追い、「**本物である**」を追わないこと（第4章）。対策：重要な事実は自分で再確認、根拠を出させる、材料を与えて「材料だけに基づいて答え」させる（第20章 RAG へ通じる）。
- 滑らかで自信満々 ≠ 信頼できる――これが最も頭に刻むべき、直感に反する判断。
- **知識 vs 記憶**：知識は**パラメータに焼き付けられ**、口をついて出るが、**知識のカットオフ（knowledge cutoff）** がある。記憶は**今回の会話**の中だけに生き、閉じれば忘れる。
- モデル本体には**会話をまたいだ記憶がない**。あなたが見る Claude Projects、ChatGPT の Memory、プロジェクト文脈ファイル、アカウントの好み、企業ナレッジベースなどは、**製品レイヤーが毎回情報をコンテキストに入れ直している**のであって、モデル本体が覚えたのではない――公式ドキュメントを参照。
- 一文で締めると：AIには「真実の倉庫」も「長期記憶の倉庫」もなく、あるのは**パラメータの中の知識（締め切りあり）＋今回机の上のコンテキスト**だけ。間違えるのも忘れるのも、すべてここに由来する。

第1部はここまでで読み終わりです――モデルが「何であり、どう動くか」の直感が身につきました。次の部からは、**どう使いこなすか**へ転じます。はじめての本格的な対話から、「自分が何をほしいか」をはっきり伝えるプロンプトの腕前まで。

## 小テスト

> 全 6 問、概念・誤解・場面・操作の四類型をカバーします。答えと解説は各問の下の引用ブロックにあります――まず自分で考えてから、照らし合わせてください。

1. **[基礎・概念]** 「ハルシネーション（hallucination）」が指すのは？
   - A. モデルがフリーズして固まり、動かなくなること
   - B. モデルが存在しない、または誤った内容を、いかにもそれらしく、自信たっぷりに語ること
   - C. モデルがあなたの会話を外部に漏らすこと
   - D. モデルの答えが遅すぎること
   > **答え：B。** ハルシネーションの要点は「まったく破綻なく、しかもやけに自信満々に間違える」こと――成り立ちは「本物らしい」を追い「本物である」を追わず、照合できる事実の倉庫がなく、知らないときは言語の惰性に沿って一つこしらえることです。A・D は性能の問題、C はプライバシーの問題（第37章）で、どれもハルシネーションではありません。

2. **[基礎・概念]** 次のうち、「知識」と「記憶」を正しく区別しているのは？
   - A. 知識は今回の会話の中だけに生き、記憶はパラメータに焼き付けられている
   - B. 知識はパラメータに焼き付けられ口をついて出る（ただし締め切りあり）。記憶は今回の会話の中だけに生き、閉じれば忘れる
   - C. 両者は同じものだ
   - D. 知識も記憶も自動で最新に更新される
   > **答え：B。** 知識は学習由来で長期的にあるが「知識のカットオフ」がある。記憶は今回のコンテキストウィンドウに支えられ、閉じる／押し出されれば消えます。A は両者を逆に言っています。D は、知識は自動更新されない（締め切りあり）、記憶はなおさら会話をまたいで残らない、という点で間違いです。

3. **[応用・誤解]** AIがある答えを返し、典拠を引いて口ぶりが断定的でした。「こんなに滑らかで自信があるのだから、きっと信頼できる」と言う人がいます。どこが間違い？
   - A. 間違っていない、自信は信頼性を意味する
   - B. 滑らかで自信満々なのは「滑らかに続けている」ことを示すだけで真偽とは無関係。重要なこと、口をついて出した事実ほど、自分で再確認する
   - C. 断定的なのはインターネットで調べたからだ
   - D. ソースを引いたなら必ず本物だ
   > **答え：B。** これは最も払拭すべき直感です――滑らかなほど警戒すべきで、「滑らかさ」こそハルシネーションが本物らしく見える表れだからです。C は仮定できません（基本的にインターネットには接続していないものと考える）。D も危険で――「ソース」さえでっち上げうるので、根拠を出させた**うえで自分で確かめる**必要があります。

4. **[応用・誤解]** 「私が使う製品には『記憶』機能があり、私の好みを覚えられる。だからモデル本体は私を覚えている。」どこが間違い？
   - A. 完全に正しい、モデルがあなたを覚えた
   - B. それは**製品レイヤー**が毎回、関連情報を自動で今回の会話のコンテキストに入れ直しているのであって、モデル本体に記憶が生えたのではない
   - C. このモデルが他のモデルより高級だという証拠だ
   - D. あなたの資料を永久にパラメータに保存したという証拠だ
   > **答え：B。** 肝心なのは二つの層を分けることです。モデル本体には会話をまたいだ記憶がなく、「記憶」機能は製品レイヤーが毎回代わりに情報をコンテキストへ入れ直すもの（第3章の机）です。AIが覚えていられるのは、永遠に製品が入れ直すと決めたその部分だけ。具体的にどう覚え、削除できるかは、**公式ドキュメントを参照**。D はさらに誤解で――製品レイヤーの記憶はモデルのパラメータを書き換えません。

5. **[基礎・場面]** 「先月発表されたばかりのある新製品」の詳細を知りたくて尋ねたら、AIがすらすらと答えました。最も無難なやり方は？
   - A. 詳しく言っているので、そのまま信じる
   - B. 「知識のカットオフ」より後のことを根拠なくでっち上げている（ハルシネーション）可能性を警戒する。インターネットで調べさせるか、公式資料を貼り付け、重要な点は自分で照合する
   - C. 何度か尋ねて、答えが一致すれば必ず正しい
   - D. 温度を上げてもう一度答えさせる
   > **答え：B。** 「最近起きたこと」はまさに知識のカットオフの盲点で、いかにもそれらしくでっち上げる可能性が高いです。インターネットで調べさせる（第19章、調べた結果も再確認）か、最新資料を読み込ませて材料に基づいて答えさせます。A はハルシネーションの罠にはまります。C は無効（でっち上げも繰り返し一致させられる）。D は的外れで、温度が高いほど気まぐれになりやすいだけです（第8章）。

6. **[基礎・操作／観察]** わざと**やや珍しく具体的な**事実を尋ね（ある少数派の本の出版年、ある法令の具体的条文など）、「出典を出して」と要求します。次に、あなたの手元の信頼できる資料を一つ貼り付けて、「この資料だけに基づいて」同じ問いに答えさせます。二回を比べてください。
   > **観察できるはず：** 材料を与えないと、いかにもそれらしいのに裏づけのない「出典」をでっち上げるかもしれません――これがハルシネーションで、「自信満々 ≠ 信頼できる」を裏づけます。材料を与え、材料だけに基づいて答えさせたあとは、正確さがふつう明らかに上がります。これがまさに RAG（第20章）の核心となる考え方です――正しく覚えていることに賭けるより、正しい資料をその場で与えるほうがよいのです。自分の手で一度比べれば、AIが口をついて出した「事実」をもう盲信しなくなります。
