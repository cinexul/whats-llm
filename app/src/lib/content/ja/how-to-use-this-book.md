この本は **基本的なパソコン操作はできるが、AI やプログラミングの背景がない一般の人** に向けています。メールの送受信ができ、ブラウザやオフィスソフトが使えるなら、十分に読みこなせます。**プログラミングができる必要はありません**。「実際に手を動かしてプロジェクトを作る」章でも、コマンドを暗記させるのではなく、平易な言葉で AI に指示する形でご案内します。

## 一、この本があなたを連れて行きたいところ

読み終えると、浅いところから深いところへ、四つのものが手に入ります。

| | 学べること | 対応する部 |
| --- | --- | --- |
| ① 分かる | 大規模モデルとは何か、なぜ言葉を続けられるのか、なぜ間違うのか――名詞の山に脅かされなくなる | 第一部 |
| ② 使える | AI を日常・学習・仕事に使い、的確に尋ね、正しく食わせ、振り回されない | 第二部 |
| ③ 見通せる | モデル/製品/API/Agent、RAG、ローカルモデル……一枚の地図で道具の全体像をつかむ | 第三部 |
| ④ 作れる | **Claude Code と Codex** という二つの主流ツールで、自分のプロジェクトを作る | 第四・五部 |

最後に第六部があり、**リスクと境界** を語ります――何を AI に食わせてよく、何を絶対に食わせてはいけないか、どうすればお金を燃やさず、地雷を踏まず、責任を持って使えるか。

> **ポイント:** これは「AI 活用テクニック大全」ではなく、あなたに **AI がどういうものかを本当に理解してもらう** ための本です。そうすれば「いつ信じてよく、いつ自分で関所を守るべきか」を自分で判断できるようになります。原理を理解すれば、テクニックは自然とついてきます。

## 二、二つの読み方、必要に応じて

最初から最後まで読み通す必要はありません。あなたに合った一本の線を選んでください。

- **「使えれば十分」という方**:**第一・二部**(基礎 + 日常の使い方)を読み、それから **第六部**(リスク)に飛びます。これで AI をすらすら、安定して使えるようになります。
- **「手を動かして何か作りたい」という方**:上の基礎の上に、**第三・四・五部**(道具の全体像 → プログラミング実践 → 六つの実践プロジェクト)を続けて読み、一歩ずつ個人サイト、文書ツール、議事録アシスタントを作り、さらには自分のプログラムに AI をつなぐところまで進みます。

どちらの線でも、**第一部は飛ばさない** ことをお勧めします――そこは本書全体の土台で、後のすべての「なぜこう使うのか」がその上に建っているからです。

## 三、付属の使い方

- **各章末の「小テスト(クイズ)」**:6 問で、**概念問題・誤解問題・シナリオ問題・操作問題** の四種類に分かれます。答えと解説は問題の下にあります――正誤だけ見るのではなく、解説で語る「なぜ間違いか、どんな人が犯しがちな間違いか」こそが要点です。
- **用語表(付録 A)**:中日英の三語対照。分からない言葉に出くわしたら、いつでも引いてください。
- **プロンプトテンプレート(付録 B)**:日常・仕事・学習・プログラミングなどの場面の「そのまま埋めて使える」テンプレート。
- **中日バイリンガル**:本書には中国語版と日本語版があり、リーダー上部の「**中文 / 日本語**」を押せばワンタッチで切り替えられ、内容は一対一で対応します。本文中の重要な用語は **初出時に括弧で英語を添えます**(例:コンテキストウィンドウ（context window）)。将来、英語の資料を調べるときに役立ちます。
- リーダーはさらに **左側の目次からの移動、全文検索、文字へのマーカーやメモ** にも対応し、どこまで読んだか、何に印を付けたかは自動で保存されます。

## 四、本書を貫く一つの心構え

もしこの本で二つの言葉だけ覚えてもらえるとしたら、それはこの二つです――

1. **AI は非常に有能だが、「もっともらしい顔で間違える」。** 追い求めているのは「本物らしく聞こえる」ことであって、「確かに本物である」ことではありません。だから **重要な事実、数字、引用は、いつも自分で検証** してください。AI を、賢く、便利で、けれどもときどき口から出まかせを言うアシスタントとして扱い、いつも正しい百科事典とは思わないでください。
2. **AI は既定では「記憶がない」。** あなたが目にする「自分を覚えている」は、その大半が **製品レイヤーが毎回あなたの代わりに情報を食わせ直している** のであって、モデル本体が本当に覚えたわけではありません(この点は第3・9章で徹底的に語ります)。これを理解すれば、「どうして忘れたの」「どうしてまた覚えているの」という多くの戸惑いが、すべて解けます。

> **ポイント:** 「AI は間違える、記憶がない」というこの二点を携えて後の各章を読めば、本書の方法と注意の数々が、ほとんどすべて「この二つの性質とどう付き合い、AI を上手に安定して使うか」を教えていることに気づくはずです。

## まとめ

- これは一般の人に向けた **大規模モデルの基礎 + AI ツールの実践** マニュアルで、**プログラミングの基礎は不要** です。
- 四つの目標:**分かる → 使える → 道具の全体像を見通せる → Claude Code と Codex でプロジェクトを作る**、そして最後にリスクの境界を守る。
- 二つの読み方:使えれば十分なら「第一・二・六部」、手を動かして作りたいなら続けて「第三・四・五部」。
- 各章の **小テスト**、**用語表**、**プロンプトテンプレート**、そして上部の **中日バイリンガル切り替え** を活用する。
- 全編を通じて二つの言葉を忘れずに:**AI は自信ありげに間違える(重要なものは自分で照合)**、**AI は既定では記憶がない(「覚えている」の多くは製品レイヤーが食わせ直しているもの)**。

準備はよいですか? それでは第1章――「『おしゃべりできる AI』から始めよう」――から始めましょう。
