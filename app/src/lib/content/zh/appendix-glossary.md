## 基础概念

**人工智能**（人工知能 ・ Artificial Intelligence (AI)）  
让机器去做"原本需要人的智力"的事，是个很大的总称。下象棋、识别图片、过滤垃圾邮件都算 AI。  
*常见误区：*常被当成"无所不能的电脑大脑"；实际它是一大类技术的统称，大模型只是其中一个分支。  

**机器学习**（機械学習 ・ Machine Learning (ML)）  
不靠人一条条写规则，而是让程序从大量数据里自己"找规律"的一种 AI 做法。是深度学习和大模型的基础。  
*常见误区：*常以为是机器"自己想明白了"；实际它只是从数据里统计出规律，并不理解含义。  

**深度学习**（深層学習 ・ Deep Learning (DL)）  
机器学习的一种，用很多层的"计算网络"（神经网络）来学习。今天的大模型、图片生成、语音合成大多基于它。  

**神经网络**（ニューラルネットワーク ・ Neural Network）  
一种受大脑神经元启发的计算结构，由很多简单单元层层连接而成。深度学习就是用很多层的神经网络。  
*常见误区：*常以为它"像人脑一样思考"；实际它只是借了个比喻，内部全是数值计算，和真神经无关。  

**大语言模型**（大規模言語モデル ・ Large Language Model (LLM)）  
从海量文本里学到"词与词怎么接续"的 AI 模型，是 Claude、ChatGPT 等产品的内核。  
*常见误区：*常被当成"无所不知的电脑大脑"；实际它只是擅长处理语言的一类模型，也会一本正经地说错。  

**模型**（モデル ・ Model）  
训练之后得到的那一大堆数值（参数）的集合，加上运行它的方法。它不是普通那种"一条条写好规则"的程序。  
*常见误区：*常以为模型是一个大数据库或一堆 if 判断；实际它是训练学出来的参数，回答是现算的，不是查出来的。  

**生成式AI**（生成AI ・ Generative AI）  
能"造出"新内容（文字、图片、代码、音频等）的 AI，而不只是做判断或分类。大模型是其中最常见的一种。  

**自然语言处理**（自然言語処理 ・ Natural Language Processing (NLP)）  
让计算机处理人类语言（理解、翻译、总结、问答等）的技术领域。大模型是这个领域近年最有力的工具。  

**人工通用智能**（汎用人工知能 ・ Artificial General Intelligence (AGI)）  
一个设想中的目标：像人一样能胜任各种任务的 AI。今天的大模型还不是 AGI，这个词更多出现在讨论和愿景里。  
*常见误区：*常把现在的聊天 AI 当成 AGI；实际它们只是擅长语言的专用模型，离"什么都会"还很远。  

**聊天机器人**（チャットボット ・ Chatbot）  
能用对话方式跟人来回交流的程序。今天厉害的聊天机器人内核大多是大模型，但"聊天机器人"这个词比大模型更老、更宽。  

**算法**（アルゴリズム ・ Algorithm）  
解决某个问题的一套固定步骤或规则。大模型并不是一条条写死的算法，而是从数据里学出来的，这正是它和传统程序的区别。  
*常见误区：*常把大模型当成"一套写好的算法/规则"；实际它的行为来自训练学到的参数，没人能逐条写出它"为什么这么答"。  

**计算机视觉**（コンピュータビジョン ・ Computer Vision）  
让计算机"看懂"图片和视频的 AI 领域（如识别人脸、读路标）。它和处理语言的大模型是 AI 下的不同分支，用来对照理解"AI 不只有大模型"。  

## 模型与训练

**参数**（パラメータ ・ Parameter）  
模型内部那一大堆"可调的数字"，训练就是不断微调它们。语言的规律就被"压"进了这些数字里。常说的"几十亿参数"指的就是它的数量。  
*常见误区：*常以为参数越多就一定越聪明；实际参数量只是规模的一个侧面，数据质量、训练方法同样关键。  

**权重**（重み ・ Weight）  
参数里最主要的一类，表示神经网络中"某个连接有多重要"。训练的过程基本就是在反复调整这些权重。  

**训练**（学習（トレーニング） ・ Training）  
给模型看大量数据、不断调整它内部参数的过程，目的是让它的输出越来越接近期望。训练做完才得到能用的模型。  

**预训练**（事前学習 ・ Pre-training）  
造大模型的第一阶段：让它读海量文本，反复练习"猜下一个词"，从而把语言的规律学进参数里。这一步最费时费钱。  

**微调**（ファインチューニング ・ Fine-tuning）  
在预训练好的模型上再用较小的数据集"补课"，让它更擅长某类任务或更会按要求回答。是对齐的常用手段之一。  
*常见误区：*常以为微调=把新知识"教"给模型当事实存起来；实际它更多是在调"行为和风格"，喂私有资料更适合用 RAG。  

**指令微调**（指示チューニング ・ Instruction Tuning）  
一种专门的微调，用大量"指令—好回答"的例子来教模型听懂并执行人的指示，让它从"续写机器"变成"会照做的助手"。  

**对齐**（アライメント ・ Alignment）  
用各种方法把模型的行为调到"有用、诚实、无害"，让它的回答符合人的意图和价值观。没对齐的模型容易乱说。  
*常见误区：*常以为对齐后模型就绝对安全可靠；实际对齐只能降低风险，还会带来"过度谨慎、动不动就拒答"的副作用。  

**人类反馈强化学习**（人間のフィードバックによる強化学習（RLHF） ・ Reinforcement Learning from Human Feedback (RLHF)）  
对齐的一种主流做法：让人给模型的多个回答打分排序，再用这些偏好去训练模型，使它更愿意给出人喜欢的回答。  

**训练数据**（学習データ ・ Training Data）  
用来训练模型的那些文本、代码等材料。数据的质量、范围和偏见，会直接影响模型最终的能力和毛病。  
*常见误区：*常以为模型把训练数据原文存了下来；实际它学的是其中的规律，原文并没被整篇保存在模型里。  

**知识截止**（知識カットオフ ・ Knowledge Cutoff）  
模型训练数据的"最后期限"。这个时间点之后才发生的事、才出的新版本，模型本身并不知道。  
*常见误区：*常以为 AI 永远知道最新消息；实际除非额外联网或喂资料，它对截止之后的事一无所知，还可能瞎编。  

**基础模型**（基盤モデル ・ Foundation Model / Base Model）  
在海量通用数据上训练出来的"打底"大模型，本身能力很广，可以再经过微调、对齐变成各种产品。  
*常见误区：*常把基础模型和成品聊天产品混为一谈；实际基础模型还没"调教好对话礼仪"，直接用往往不听话。  

**蒸馏**（蒸留 ・ Distillation）  
用一个又大又强的模型当"老师"，去训练一个更小的"学生"模型，让小模型用更低成本逼近大模型的表现。  

**模型规模**（モデル規模 ・ Model Scale / Model Size）  
衡量模型有多"大"，主要看参数数量和训练数据量。规模越大通常越强，但也更慢、更贵。  

**模型版本/代际**（モデルのバージョン・世代 ・ Model Version / Generation）  
同一系列模型会不断推出新版本（如更新换代），能力和定价随之变化。  

**上下文长度（训练时）**（学習時の文脈長 ・ Context Length）  
模型在设计和训练时定下的"一次最多能处理多少 token"的能力上限，对应到使用时就是上下文窗口的大小。  

**规模法则**（スケーリング則 ・ Scaling Laws）  
一个经验规律：模型、数据、算力越大，效果往往越好。这解释了为什么各家拼命把模型做大，但它不是无限免费的午餐。  
*常见误区：*常以为"只要堆大就一定更好"；实际收益会递减，且数据质量、训练方法同样决定上限。  

## 运作原理

**token**（トークン ・ Token）  
大模型眼里文字的最小单位，可能是一个词、半个词或一个字。模型不是按"字"而是按 token 来读和算的。  
*常见误区：*常以为一个 token 就是一个字或一个词；实际它是模型自己切出来的片段，中英文、长词短词切法都不同。  

**分词**（トークン化（分かち書き） ・ Tokenization）  
把一段文字拆成一个个 token 的过程。不同模型的切法不一样，这也是为什么同样一句话在不同模型里 token 数不同。  

**上下文窗口**（コンテキストウィンドウ ・ Context Window）  
模型一次能"同时看到"的 token 上限，整段对话都得塞进这块"临时工作台"。超了，早先的内容就会被挤掉。  
*常见误区：*常以为它是 AI 的长期记忆；实际它只是这次对话的临时容量，关掉或塞满就忘，且越满越慢越贵。  

**下一个词预测**（次トークン予測 ・ Next-Token Prediction）  
大模型干的核心就这一件事：根据前面的内容，猜"下一个最可能出现的 token"，一个个接下去，连成整段话。  
*常见误区：*常以为 AI 是先想好答案再说出来；实际它是一个 token 一个 token 往外蹦，边写边猜，并没有事先的"答案"。  

**概率分布**（確率分布 ・ Probability Distribution）  
模型在挑下一个 token 时，会给每个候选词算一个概率（比如"东京 92%、江户 0.8%"），再从中挑一个。  

**采样**（サンプリング ・ Sampling）  
从那张"候选词概率表"里挑出实际要输出的词的过程。挑法可以偏稳（总挑高概率）也可以偏随机，由温度等参数控制。  

**温度**（温度（temperature） ・ Temperature）  
一个控制"输出随机性"的旋钮。调低更稳、几乎每次一样；调高更发散、更有创意但也更容易跑偏。  
*常见误区：*常以为温度高=更聪明；实际它只改变"敢不敢挑冷门词"，和对错无关，太高反而容易胡来。  

**Top-p采样**（Top-p（核サンプリング） ・ Top-p (Nucleus) Sampling）  
另一个控制随机性的旋钮：只在"概率加起来够高的那一小撮候选词"里挑，过滤掉太冷门的选项。常和温度配合使用。  

**推理强度/扩展思考**（拡張思考（推論の深さ） ・ Reasoning Effort / Extended Thinking）  
让模型在给答案前先在内部多"想几步"（生成思考草稿），用更多算力换更靠谱的结果。适合难题，简单问题没必要。  
*常见误区：*常以为"想得越久答案一定越对"；实际它能提升复杂推理，但也更慢更贵，且仍可能出错，简单任务反而浪费。  

**向量**（ベクトル ・ Vector）  
一串数字组成的"坐标"。把词、句子变成向量后，机器就能用数学去比较它们的远近，也就是意思的相近程度。  

**嵌入**（埋め込み（エンベディング） ・ Embedding）  
把一段文字转成一串能表达"意思"的数字（向量）的做法。意思相近的文字，向量也会靠得近，是搜索和 RAG 的基础。  
*常见误区：*常以为 embedding 存的是原文；实际它存的是"意思的坐标"，看不出原句，但能算出谁和谁意思相近。  

**向量空间**（ベクトル空間 ・ Vector Space）  
把所有词、句子的向量摆在一起形成的"多维空间"。意思相近的就挨得近，著名的"国王-男人+女人≈女王"就发生在这里。  

**相似度**（類似度 ・ Similarity）  
衡量两段文字"意思有多接近"的数值，通过比较它们的向量算出来。搜索和 RAG 靠它找出"最相关"的资料。  

**注意力机制**（アテンション機構 ・ Attention Mechanism）  
模型读一句话时，决定"每个词该多关注其它哪些词"的能力。说白了就是会划重点，这让它能抓住长句里的关键联系。  
*常见误区：*常以为注意力=模型"真的读懂了"；实际它算的是词与词在统计上的相关，是数学加权，不是理解。  

**自注意力**（自己注意（セルフアテンション） ・ Self-Attention）  
注意力机制的核心形式：让一句话里的每个词都去看同一句话里的其它词，从而把上下文关系编织进来。Transformer 的关键部件。  

**Transformer**（トランスフォーマー ・ Transformer）  
当前几乎所有主流大模型的底层架构，核心是注意力机制。可以理解为大模型的"发动机型号"，知道有这么个东西即可。  
*常见误区：*常把 Transformer 当成某个具体产品；实际它是一种技术架构，Claude、GPT、Gemini 等都建在它之上。  

**编码器/解码器**（エンコーダ・デコーダ ・ Encoder / Decoder）  
Transformer 里的两类部件：编码器负责"读懂输入"，解码器负责"生成输出"。聊天类大模型主要用解码器那一路。  

**知识**（知識 ・ Knowledge (parametric)）  
训练时焊进参数里的那些"常识和事实"，模型张口就来，不依赖你这次给的材料。但它截止之后的事不在其中。  
*常见误区：*常把"模型知道"当成"模型存了原文可查"；实际那是训练学到的统计印象，可能记串、记错。  

**记忆**（記憶（文脈上の記憶） ・ Memory (in-context)）  
只活在"这次对话"里的临时记忆，全靠上下文窗口撑着。关掉对话或被挤出窗口，它就忘得一干二净。  
*常见误区：*常以为 AI 会像人一样长期记住你；实际默认情况下它没有跨对话的记忆，下次见面是"失忆"状态。  

**幻觉**（ハルシネーション ・ Hallucination）  
模型把不存在或错误的内容，说得有模有样、很自信。因为它追求的是"像真的"，不是"是真的"。  
*常见误区：*常以为说得越流畅自信就越可靠；实际越顺溜越要警惕，重要事实必须自己复核。  

**涌现能力**（創発的能力 ・ Emergent Abilities）  
模型规模大到一定程度后，"突然"表现出小模型没有的新能力（如做多步推理）。像水到一定温度才沸腾。  
*常见误区：*常把涌现说成 AI"觉醒"或有了意识；实际它只是规模带来的能力跃升，内部还是数值计算。  

**多模态**（マルチモーダル ・ Multimodal）  
模型不只处理文字，还能处理图片、音频甚至视频等多种"模态"的信息。比如看图说话、读截图。  

**确定性vs随机性**（決定論的・確率的な出力 ・ Determinism vs Randomness）  
因为生成带随机性，同一个问题问多次，答案可能每次都略有不同。"上次对了"不代表"这次也对"。  
*常见误区：*常以为同样的输入一定给同样的输出；实际默认带随机性，结果会变，所以重要输出都要逐次验证。  

## 使用方法

**提示词**（プロンプト ・ Prompt）  
你发给 AI 的那段话或指令。你给的背景越清楚，它的回答通常越好——"输入的质量决定输出的质量"。  
*常见误区：*常以为它能猜中你没说的意图；实际它读不懂你的心，背景、要求、想要的格式都得明说。  

**提示工程**（プロンプトエンジニアリング ・ Prompt Engineering）  
摸索"怎么把话说清楚、把要求摆明白"，让 AI 更好地给出你想要的结果的一套方法。不神秘，本质就是会提需求。  

**系统提示**（システムプロンプト ・ System Prompt）  
在对话最前面设定 AI"身份和总规矩"的一段指令，比如"你是一位严谨的法律顾问"。它会影响整段对话的风格和边界。  

**零样本**（ゼロショット ・ Zero-shot）  
不给任何例子，直接让 AI 完成任务。简单任务通常够用，复杂或格式特殊的任务则更适合给几个示例。  

**少样本/示例**（フューショット（少数例） ・ Few-shot）  
在提示词里给几个"输入—想要的输出"示例，让 AI 照葫芦画瓢。想要固定格式或特定风格时特别有用。  

**思维链**（思考の連鎖（Chain of Thought） ・ Chain of Thought (CoT)）  
让 AI"一步步把推理过程写出来"再给结论，对数学题、逻辑题等复杂任务往往更准。  

**上下文**（文脈（コンテキスト） ・ Context）  
你这次给 AI 的所有信息（问题、背景、贴进去的资料、之前的对话）。它当下只能依据这些来回答。  

**会话/对话**（セッション（会話） ・ Session / Conversation）  
从你打开聊天到结束的一整段来回。同一段对话里 AI 记得前文；换一段新对话，它就从零开始。  

**多轮对话**（マルチターン会話 ・ Multi-turn Conversation）  
围绕一个话题来回聊很多轮。聊得越久，上下文越占地方，也越容易跑题或"越聊越笨"，必要时该开新对话。  
*常见误区：*常以为聊得越久 AI 越懂你；实际话题混杂、上下文塞满后反而更容易出错，一个话题一个对话更稳。  

**摘要/总结**（要約 ・ Summarization）  
让 AI 把长文压成要点。要点是"基于我给的材料来总结，不要自己加料"，否则容易掺进编造。  

**结构化输出**（構造化出力 ・ Structured Output）  
要求 AI 按固定格式输出，比如表格、列表、JSON。明确说清字段和格式，结果更整齐也更好用。  

**角色设定**（ロール設定（役割付与） ・ Role / Persona Setting）  
给 AI 指定一个身份或视角（如"资深编辑""面试官"），让它的语气和侧重点贴合你的需要。  

**迭代修正**（反復的な修正 ・ Iterative Refinement）  
不指望一次到位，而是"出初稿—指出哪里不对—让它改"，一轮轮逼近你想要的结果。这是用好 AI 的常态。  

**提示词模板**（プロンプトテンプレート ・ Prompt Template）  
把常用的好指令固定成"填空模板"，下次换个内容就能复用，省得每次从头写。  

**验证/复核**（検証・裏取り ・ Verification / Fact-checking）  
对 AI 给的事实、数字、引用，自己再查证一遍。因为它会幻觉，重要内容绝不能拿来直接用。  
*常见误区：*常以为查证太麻烦可以省；实际这是用 AI 最关键的一步，省掉它等于把它的错当成你的错。  

**联网搜索/工具增强**（ウェブ検索・ツール連携 ・ Web Search / Tool-augmented）  
让 AI 在回答时去联网查最新信息或调用工具，弥补"知识截止"和记不住的问题。能查到最新内容，但查到的也要复核。  
*常见误区：*常以为开了联网答案就一定准；实际它仍可能选错来源或误读，最终还得你核对引用。  

**重新生成/换个说法**（再生成・言い換え ・ Regenerate / Rephrase）  
对不满意的回答，让它重答一次或换种方式问。因为输出带随机性、又对措辞敏感，换个说法常能得到更好结果。  

**草稿/初稿思维**（たたき台 ・ First Draft / Starting Point）  
把 AI 的产出当"初稿"而非"成品"：它帮你快速起个头，你再来判断、修改、定稿。这是最稳妥的协作心态。  
*常见误区：*常想着让 AI 一步到位直接交付；实际把它当"打草稿的实习生"、由你收尾，效果和安全都更好。  

**提示词泄露**（プロンプト漏洩 ・ Prompt Leaking）  
有人用话术诱导 AI 把你设定的系统提示或隐藏指令"吐出来"。所以别把真正的机密写进提示词里。  

## 工具生态

**模型vs产品**（モデルと製品の違い ・ Model vs Product）  
模型是底层内核（如 GPT、Claude 系列），产品是包装好给人用的东西（如 ChatGPT 网页、Claude 应用）。一个产品背后可能换用不同模型。  
*常见误区：*常把产品名和模型名当成一回事；实际 ChatGPT 是产品、GPT 是模型，二者是"车"和"发动机"的关系。  

**API**（API ・ Application Programming Interface (API)）  
让你自己的程序直接"调用"模型的入口，跳过聊天网页。"给程序接上 AI"通常就是指调 API。  
*常见误区：*常以为 API 是另一个更强的 AI；实际它只是访问同一个模型的另一种方式（程序对程序），按用量计费。  

**API密钥**（APIキー ・ API Key）  
调用 API 时用来证明"是你"的一串机密字符串，等于账户的钥匙。泄露了别人就能花你的钱。  
*常见误区：*常以为密钥贴进聊天框或代码里没事；实际它一旦外泄就可能被盗刷，绝不能公开，泄露要立刻作废重置。  

**Token计费**（トークン課金 ・ Token-based Pricing）  
用 API 通常按 token 数收费，输入和输出都算钱。聊得越长、喂的资料越多，花的也越多。  
*常见误区：*常以为只有自己打的字算钱；实际 AI 的回复、以及每轮重新带上的历史对话都算 token，长对话很烧钱。  

**SDK**（SDK ・ Software Development Kit (SDK)）  
厂商提供的一套现成代码库，帮开发者更省事地调用 API，不用自己从零拼接请求。  

**推理（运行模型）**（推論（インファレンス） ・ Inference）  
指"运行一个已经训练好的模型来产出结果"，也就是你每次提问、它生成回答的那个计算过程。和"训练"是两回事。  
*常见误区：*常把这里的"推理"理解成"逻辑推理"；实际它指的是"运行模型出结果"这件事，是个工程术语。  

**智能体**（エージェント ・ Agent）  
不只回答问题，还能"动手"的 AI：给它目标，它会自己调用工具、按"调查→执行→检查"的循环往前推进。  
*常见误区：*常以为 Agent 就是更聪明的聊天机器人；实际它的关键是"能真的动手做事"，能力和风险都来自这一点。  

**工具调用**（ツール呼び出し（ファンクションコーリング） ・ Tool Use / Function Calling）  
让模型在回答时"调用外部工具"（查数据库、算数、发请求等）的机制。这是 Agent 能动手的技术基础。  
*常见误区：*常以为模型自己就能上网、算数；实际它得通过被授予的工具去做，没接工具时只能凭嘴说。  

**Claude**（Claude ・ Claude）  
Anthropic 公司的 AI 产品/模型系列名（含 Opus、Sonnet、Haiku 等）。它是大模型的一个代表，不是"全部 AI"。  

**Claude Code**（Claude Code ・ Claude Code）  
一款在终端（命令行）里运行的编程智能体，能读项目文件、改代码、跑命令。可理解为"装上了手脚的 Claude"。  

**ChatGPT**（ChatGPT ・ ChatGPT）  
OpenAI 公司的聊天产品，内核是 GPT 系列模型。是很多人接触的第一个大模型产品。  

**Codex**（Codex ・ Codex）  
OpenAI 这条线的编程/编码智能体方向的产品。和 Claude Code 类似，都属于"会动手写代码"的工具。  

**Gemini**（Gemini ・ Gemini）  
Google 公司的 AI 产品/模型系列名。也是主流大模型之一，常和 Claude、ChatGPT 放在一起比较。  

**Copilot**（Copilot ・ Copilot）  
泛指嵌在编辑器/办公软件里的 AI 助手（如代码补全、文档助手）。这里只讲它作为'辅助副驾'的概念。  

**检索增强生成**（検索拡張生成（RAG） ・ Retrieval-Augmented Generation (RAG)）  
先从你的资料库里"查"出相关内容，再把它塞给模型让它"基于这些资料"回答。用来绕开知识截止、减少幻觉、用上私有资料。  
*常见误区：*常以为 RAG=把资料"训练"进模型；实际它是临时查出来喂进上下文，没改模型本身，资料更新也无需重训。  

**向量数据库**（ベクトルデータベース ・ Vector Database）  
专门存"文本向量（embedding）"并能快速找出"最相近内容"的数据库。是 RAG 里负责"检索"那一步的常用部件。  

**知识库**（ナレッジベース ・ Knowledge Base）  
你专门整理好、供 AI 检索使用的一批资料（文档、网页、手册等）。RAG 就是从这里取料。  

**本地模型**（ローカルモデル ・ Local Model）  
跑在你自己电脑（或自家服务器）上、不经过云端的大模型。好处是隐私和离线，代价是要硬件、通常没云端模型强。  
*常见误区：*常以为本地模型一定不如云端、或一定更安全；实际它确实更私密，但能力、速度受你的硬件限制，强弱要看具体型号。  

**开源模型/开放权重**（オープンソースモデル・オープンウェイト ・ Open-source / Open-weight Model）  
把模型权重公开、允许大家下载自己跑的模型。它让本地部署成为可能，常用于离线或隐私敏感场景。  
*常见误区：*常把"开放权重"等同于"完全开源可商用"；实际不同模型的授权条款差别很大，用前要看清许可证。  

**Ollama**（Ollama ・ Ollama）  
一个让你在本地电脑上方便地下载和运行开源大模型的工具，命令简单，常被新手用来入门本地模型。  

**LM Studio**（LM Studio ・ LM Studio）  
一个带图形界面、方便在本地下载和运行开源大模型的桌面应用，适合不想敲命令的人尝试本地模型。  

**量化**（量子化 ・ Quantization）  
把模型里的数字用更"粗"的精度存储，让模型变小、跑得动普通电脑，代价是质量略有损失。本地跑大模型常用这招。  
*常见误区：*常以为量化会让模型"变笨很多"；实际适度量化通常只略降质量，却能大幅降低对硬件的要求。  

**Skills/技能**（スキル ・ Skills）  
把"一套做某件事的步骤和相关文件"打包成可复用的"技能"，需要时才加载，不占平时的上下文。可理解为给 AI 的即插即用插件包。  

**模型上下文协议**（モデルコンテキストプロトコル（MCP） ・ Model Context Protocol (MCP)）  
一种让 AI 应用统一连接外部工具和数据源的开放标准，常被比作"AI 世界的 USB 接口"，插上就能用各种外设。  
*常见误区：*常以为 MCP 是某个产品或服务器；实际它是一套"接口规范"，具体功能由各个 MCP 服务器提供。  

**MCP服务器**（MCPサーバー ・ MCP Server）  
按 MCP 标准提供某种工具或数据接口的程序（比如连上你的日历、数据库）。AI 作为客户端去连接它来取用能力。  

**插件**（プラグイン ・ Plugin）  
给 AI 产品额外加装能力的小模块，让它能连上某个外部服务或多出某种功能。是"扩展能力"的通俗说法。  

**多模态输入**（マルチモーダル入力 ・ Multimodal Input）  
除了打字，还能给 AI 上传图片、文件、音频等让它一起处理，比如发张截图让它解释。  

**云端模型**（クラウドモデル ・ Cloud Model / Hosted Model）  
跑在厂商服务器上、通过网络访问的大模型（绝大多数聊天产品都是这样）。好处是省心、能力强，代价是要联网、数据要出本机。  
*常见误区：*常以为"云端=不安全、本地=万无一失"；实际两者各有取舍，关键看你的数据有多敏感、需求是什么。  

**上下文工程**（コンテキストエンジニアリング ・ Context Engineering）  
精心安排"该把哪些信息、按什么顺序放进上下文"的功夫。给得准比给得多更重要，是用好 RAG、长任务和项目上下文的核心思路。  

**嵌入模型**（埋め込みモデル ・ Embedding Model）  
专门把文字转成向量（embedding）的模型，和负责生成对话的大模型常常是两个不同的模型。RAG 用它来给资料"编坐标"。  

**提示词缓存**（プロンプトキャッシュ ・ Prompt Caching）  
把反复用到的同一段上下文"缓存"起来，下次复用时更快也更省钱。对反复带相同长前缀的场景很有用。  

**应用/封装层**（アプリ・ラッパー層 ・ Application / Wrapper Layer）  
在模型或 API 之上做出的具体应用（网页、插件、行业工具）。同一个底层模型上面可以长出无数应用，这层决定了你实际怎么用它。  
*常见误区：*常把"套壳应用"一律贬低；实际好的封装在提示、流程、数据上做了很多工，体验差别可以很大。  

## AI编程与Agent

**编程智能体**（コーディングエージェント ・ Coding Agent）  
专门用来写代码、改文件、跑命令的 Agent（如 Claude Code、Codex）。把它当"很能干但刚入职的同事"，最终责任仍在你。  
*常见误区：*常以为它能全自动交付、不用管；实际它会犯错、会理解偏，必须由人看 diff、跑测试、做验收。  

**终端/命令行**（ターミナル（コマンドライン） ・ Terminal / Command Line (CLI)）  
一个靠"打字命令"来操作电脑的黑底窗口。很多编程智能体在这里运行。看着硬核，但跟着步骤照做即可。  
*常见误区：*常以为命令行是"黑客专用、容易搞坏电脑"；实际它只是另一种操作方式，按提示一步步来并不可怕。  

**diff（差异）**（差分（diff） ・ Diff）  
把'改之前'和'改之后'并排标出来，让你一眼看清 AI 到底动了哪几行。是验收改动的基础。  
*常见误区：*常以为看 diff 是程序员才需要的；实际它就是"改了啥"的对照表，不懂代码也该养成先看 diff 再放行的习惯。  

**版本控制/Git**（バージョン管理（Git） ・ Version Control / Git）  
一套记录文件每次改动历史的系统。有了它，AI 改坏了也能一键回到之前的版本，是"敢让 AI 动手"的安全网。  
*常见误区：*常以为 Git 只是程序员的工具；实际它给你"随时反悔"的能力，是让 AI 放手干而不失控的关键保障。  

**提交/commit**（コミット ・ Commit）  
把当前这批改动"存一个档"记进 Git 历史。每次提交都是一个可回到的存档点，方便日后查和退。  

**分支**（ブランチ ・ Branch）  
从主线"岔出一条平行的工作线"，在上面随便改、改坏也不影响主线。让 AI 大刀阔斧时，先开个分支更安全。  

**回滚**（巻き戻し（ロールバック） ・ Rollback / Revert）  
把文件恢复到之前某个存档点，相当于"一键反悔"。AI 改出问题时，回滚是最快的急救手段之一。  

**测试**（テスト ・ Test）  
一段用来自动检查"程序有没有按预期工作"的代码。改完跑一遍测试，能快速发现 AI 是不是把别的地方弄坏了。  

**代码评审**（コードレビュー ・ Code Review）  
在采纳改动前，由人（或另一个 AI）逐处检查代码改得对不对、好不好。AI 写得再快，评审和放行也得人来把关。  

**退化**（デグレード（リグレッション） ・ Regression）  
改了新功能，却把原本好好的旧功能弄坏了。靠把已有测试全跑一遍来及时发现。  

**CLAUDE.md/项目上下文**（CLAUDE.md（プロジェクト文脈） ・ CLAUDE.md / Project Context）  
放在项目里、AI 每次开工都会先读的一份"项目说明书"，写明技术栈、规矩、禁忌等，相当于给 AI 的常驻记忆。  
*常见误区：*常以为写得越多越好；实际写太多反而占上下文、稀释重点，应只写"真正影响它干活"的规矩。  

**许可机制**（許可（パーミッション）の仕組み ・ Permission System）  
编程智能体在"改文件、跑命令"前先向你请示、由你点头才执行的机制。把危险操作交给人确认，是它"动手但不失控"的关键。  

**计划模式**（プランモード ・ Plan Mode）  
让 AI 先把"打算怎么改"的计划列出来、你批准后再动手的模式。大改动先看计划，能省下最多返工。  

**子代理**（サブエージェント ・ Subagent）  
主 AI 临时派出去干专项活的"另一个 AI"，有自己独立的上下文，干完只把结果摘要带回来，不占主线的篇幅。  

**任务拆分**（タスク分割 ・ Task Decomposition）  
把一个大任务切成"调查→计划→执行→检查"等小步骤，一步步交给 AI。这样它更不容易跑偏，你也更好把控。  

**验收**（受け入れ確認 ・ Acceptance / Sign-off）  
最后由你确认"这次改动确实满足要求、没弄坏别的"才采纳。看不懂、没把握，就别合并——责任在人。  
*常见误区：*常以为测试通过就万事大吉；实际测试只是底线，是否真正满足需求、能否上线，仍需人来拍板。  

**工作流**（ワークフロー ・ Workflow）  
把"一问一答"升级成一套固定步骤（如调查→计划→执行→验收），让 AI 沿着流程持续推进任务又不失控。  

**集成开发环境**（統合開発環境（IDE） ・ Integrated Development Environment (IDE)）  
程序员写代码的主力软件（如 VS Code）。很多 AI 编程助手能装进它里面用。  

**仓库**（リポジトリ ・ Repository (repo)）  
一个项目所有文件和它们改动历史的"总仓库"，通常由 Git 管理。AI 干活、提交、回滚都围绕它进行。  

**拉取请求**（プルリクエスト ・ Pull Request (PR)）  
在协作平台上"申请把我这批改动并入主线"的流程，方便别人先评审再合并。AI 可以帮你写好说明并发起 PR。  

**重构**（リファクタリング ・ Refactoring）  
在不改变程序"对外表现"的前提下，整理内部结构让代码更清晰好维护。常用"原有测试还能全过"来确认没改坏。  

**调试**（デバッグ ・ Debugging）  
找出并修掉程序里的错误（bug）。编程智能体能读报错、查原因、试着改了再跑，自己转这个"试错循环"。  

**沙箱/隔离环境**（サンドボックス（隔離環境） ・ Sandbox / Isolated Environment）  
一个"出事也不会波及真实系统"的隔离运行环境。在里面让 AI 放手试，比在正式环境里安全得多。  

**急救/卡住怎么办**（トラブル時の応急処置 ・ Recovery / Troubleshooting）  
AI 卡住、越改越乱时的标准动作：中断当前、开新对话、回滚改动、重新描述问题。先止损，再排查。  

## 风险与安全

**隐私**（プライバシー ・ Privacy）  
你输入给 AI 的内容会被怎么处理、会不会被留存或用于训练的问题。涉及个人或机密信息时尤其要当心。  
*常见误区：*常以为输进去的东西"用完就没了"；实际是否留存、是否用于训练取决于产品设置与条款，且会变，要先确认。  

**数据分级**（データ分類（機密度レベル） ・ Data Classification / Tiering）  
把信息按敏感程度分几档（如公开、内部、机密、绝密），据此决定哪些能喂给 AI、哪些绝对不能。一个简单实用的自保框架。  

**提示注入**（プロンプトインジェクション ・ Prompt Injection）  
攻击者把"假指令"藏在 AI 会读到的外部内容里（网页、邮件、文档），骗它做出你没想让它做的事。  
*常见误区：*常以为风险只来自你自己打的字；实际 AI 读到的任何外部内容里都可能埋雷，接入外部数据时尤其要防。  

**越狱**（ジェイルブレイク ・ Jailbreak）  
用特殊话术诱导 AI 绕开它的安全限制、说出本不该说的内容。属于滥用行为，也是厂商持续在防的攻击。  

**敏感信息/密钥**（機密情報・シークレット ・ Sensitive Information / Secrets）  
密码、API 密钥、数据库连接串、私钥等绝不能外泄的东西。永远不要贴进聊天框或代码；万一贴了，立刻作废重置。  
*常见误区：*常以为贴进 AI 对话框是"私下的"；实际它可能被留存或泄露，等于把家门钥匙发到群里。  

**个人信息（PII）**（個人情報（PII） ・ Personally Identifiable Information (PII)）  
能识别到具体个人的信息，如姓名、住址、电话、身份证号、邮箱。喂给 AI 前要先确认有没有混入并做脱敏。  
*常见误区：*常以为只有身份证号才算 PII；实际姓名加单位、邮箱、定位等组合起来也能认人，都要谨慎。  

**内容审核**（コンテンツモデレーション ・ Content Moderation）  
对 AI 的输入输出做安全过滤，挡掉违法、有害或不当内容。既指厂商内置的护栏，也指你自己对生成内容把关。  

**版权**（著作権 ・ Copyright）  
AI 生成内容"能不能直接用、算谁的"涉及版权问题，规则仍在发展中。商用前尤其要谨慎核对。  
*常见误区：*常以为 AI 生成的就一定能随便商用；实际权属和可用性视具体情形和当地规则而定，别想当然。  

**署名/AI披露**（AIの利用開示（出所表示） ・ AI Disclosure / Attribution）  
在合适场合注明"这部分是 AI 生成或辅助的"，以示诚实、便于他人判断。是否必须标注要看场景与规定。  

**开源许可证**（OSSライセンス ・ Open-source License）  
开源代码的使用条款。AI 给你引入的第三方库，要确认它的许可证是否允许你这样用，否则可能违规。  
*常见误区：*常以为开源就是"随便用"；实际不同许可证限制差别很大，有的要求公开你的代码或保留声明。  

**上下文成本**（コンテキストコスト ・ Context Cost）  
上下文越长，每次回答就越慢、越贵，也更容易跑偏。所以"省上下文"往往同时省钱、提速、提质，三位一体。  
*常见误区：*常以为把所有资料一股脑都塞进去最稳；实际塞太多既烧钱又拖慢，还稀释重点，反而更容易出错。  

**额度/限流**（利用上限・レート制限 ・ Quota / Rate Limit）  
厂商对你单位时间能用多少做的限制，超了就得等或要更高档位。用量大的时候会实际遇到。  

**零数据保留**（ゼロデータ保持 ・ Zero Data Retention (ZDR)）  
一种约定：服务方不保存你的输入输出（用完即弃），常用于对隐私要求高的场景。是否可用、怎么开通要看产品。  

**数据用于训练**（学習への利用 ・ Use of Data for Training）  
指你的输入是否会被服务方拿去训练后续模型。这关系到机密会不会"流出"，使用前应确认并按需关闭。  
*常见误区：*常以为所有产品默认都拿你的数据训练（或都不拿）；实际各家政策不同、可设置且会变，必须逐个确认。  

**偏见**（バイアス ・ Bias）  
模型从训练数据里学到的不公平倾向（如对某些群体的刻板印象）。它会无意识地带进回答里，需要警惕和复核。  
*常见误区：*常以为机器"客观中立"；实际它会照单学下数据里的偏见，输出未必公平，重要场景要人来把关。  

**过度依赖**（過度の依存 ・ Over-reliance / Automation Bias）  
不假思索地相信 AI 给的答案，把它当权威。它会自信地说错，越省事的时候越容易踩坑，关键判断仍要由人来下。  
*常见误区：*常以为"AI 都说了应该没错"；实际它没有判断对错的能力，只是说得像，重要决定不能直接外包给它。  

**数据脱敏**（マスキング（匿名化） ・ Data Masking / Anonymization）  
在把资料喂给 AI 之前，先把姓名、电话、密钥等敏感部分抹掉或替换成假值。是"既想用 AI 又要保护隐私"的常用折中。  

**团队使用规范**（チーム利用ガイドライン ・ Team Usage Policy）  
团队事先商定"什么能喂、什么不能喂、谁来审、密钥怎么管"的一份规矩。把个人习惯升级成共识，避免有人无意闯祸。  

**幻觉引用**（架空の出典 ・ Fabricated Citation）  
幻觉的一种典型形式：AI 编出看起来正规、其实根本不存在的文献、链接或法条。引用务必逐条核实真伪。  
*常见误区：*常以为"它给了出处就可信"；实际出处本身就可能是编的，链接和文献名都要点开查证。  

**安全护栏**（ガードレール ・ Guardrails）  
给 AI 系统加的各种"不许越界"的限制（拒答违法请求、过滤危险输出等），既靠厂商内置，也可由使用方自己加。  

## 术语缩写

**AI（缩写）**（AI（略語） ・ AI = Artificial Intelligence）  
人工智能的英文缩写。泛指让机器完成需要智力的任务的各类技术。  

**ML（缩写）**（ML（略語） ・ ML = Machine Learning）  
机器学习的英文缩写。让程序从数据里自己学规律的一类方法。  

**DL（缩写）**（DL（略語） ・ DL = Deep Learning）  
深度学习的英文缩写。用多层神经网络来学习的机器学习方法。  

**LLM（缩写）**（LLM（略語） ・ LLM = Large Language Model）  
大语言模型的英文缩写，是 Claude、ChatGPT 等产品的内核。  

**NLP（缩写）**（NLP（略語） ・ NLP = Natural Language Processing）  
自然语言处理的英文缩写，指让计算机处理人类语言的技术领域。  

**API（缩写）**（API（略語） ・ API = Application Programming Interface）  
应用程序接口的英文缩写。让你的程序直接调用模型的入口。  

**RAG（缩写）**（RAG（略語） ・ RAG = Retrieval-Augmented Generation）  
检索增强生成的英文缩写。先查资料再让模型基于资料作答的方法。  

**MCP（缩写）**（MCP（略語） ・ MCP = Model Context Protocol）  
模型上下文协议的英文缩写。让 AI 统一连接外部工具和数据的开放标准。  

**SDK（缩写）**（SDK（略語） ・ SDK = Software Development Kit）  
软件开发工具包的英文缩写。帮开发者更省事地调用 API 的现成代码库。  

**PII（缩写）**（PII（略語） ・ PII = Personally Identifiable Information）  
可识别个人信息的英文缩写，如姓名、住址、电话、邮箱等。  

**RLHF（缩写）**（RLHF（略語） ・ RLHF = Reinforcement Learning from Human Feedback）  
人类反馈强化学习的英文缩写，是模型对齐的一种主流做法。  

**GPU（缩写）**（GPU（略語） ・ GPU = Graphics Processing Unit）  
图形处理器的英文缩写。原本用于画图，因擅长大规模并行计算而成为训练和运行大模型的主力硬件。  
*常见误区：*常以为它只是"打游戏的显卡"；实际它是跑大模型的关键算力，本地跑模型快不快很大程度看它。  

**CLI（缩写）**（CLI（略語） ・ CLI = Command Line Interface）  
命令行界面的英文缩写。靠打字命令来操作软件或电脑的方式，很多编程智能体在这里运行。  

**GPT（缩写）**（GPT（略語） ・ GPT = Generative Pre-trained Transformer）  
一类大模型的名字缩写（生成式预训练 Transformer），是 ChatGPT 的内核模型系列。  

**JSON（缩写）**（JSON（略語） ・ JSON = JavaScript Object Notation）  
一种常见的"结构化数据"文本格式，键值成对、机器好解析。让 AI 输出 JSON，方便程序接着处理。  

**ZDR（缩写）**（ZDR（略語） ・ ZDR = Zero Data Retention）  
零数据保留的英文缩写。服务方用完即不保存你输入输出的一种约定。
