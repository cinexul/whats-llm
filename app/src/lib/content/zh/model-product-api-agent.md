刚接触 AI 的人，最先被劝退的往往不是技术，而是**名词**：Claude、ChatGPT、GPT、Gemini、Copilot、API、Agent、智能体……它们到底谁是谁、谁包含谁？同一家公司怎么又有模型又有产品，名字还差不多？

这一章给你一张**地图**。你会发现，市面上几乎所有 AI 的名词，都能塞进**四个层次**里。看懂这四层的关系，名词的迷雾就散了一大半——后面几章讲产品对比（第18章）、Agent（第19章）、给程序接 AI（第35章），都是在这张地图上展开。

## 一、先上地图：四个层次

我们用一个贯穿全章的比喻：**一位很能干的厨师**。

| 层次 | 是什么 | 厨师比喻 | 例子（**具体能力以官方文档为准**） |
| --- | --- | --- | --- |
| ① 模型（model） | 真正"会思考"的内核 | 厨师本人的脑子和手艺 | Claude、GPT、Gemini 等"模型系列" |
| ② 产品（product） | 把模型包装成人能直接用的东西 | 一家餐厅：有菜单、服务员、餐桌 | Claude（App/网页）、ChatGPT、Gemini（App） |
| ③ API（接口） | 把模型能力开放给**程序**调用的接口 | 厨师的"接单窗口"，别的店能下单 | 各家的 API / 开发者接口 |
| ④ Agent（智能体） | 给模型装上"手"和一个目标，让它自己动手跑完整个任务 | 会自己采购、备料、上菜、收拾的厨师 | Claude Code、Codex 等"智能体" |

> **要点：** 这四层不是四个互斥的东西，而是**从内到外、层层包裹**的关系。同一个模型，可以装进一个产品里给人用，也可以通过 API 给程序用，还可以被做成一个会自己干活的 Agent。先记住这句话，下面逐层拆开。

## 二、第①层 模型：会思考的"内核"

**模型**就是第一部分讲的那个东西——从海量文本里学会"怎么接话"的大语言模型（第1、2章）。它是真正"会算"的内核，但它**本身不是一个你能直接打开来用的软件**。

- 你没法"下载一个 GPT 就开始聊"，就像你没法把一位厨师的"手艺"单独拿在手里——手艺得依附在人身上、在厨房里才能做出菜。
- 各家公司一般有**一个模型系列**，里面分大小档位（更强但更慢更贵 / 更快更便宜），名字常带版本号。**具体型号和能力会频繁更新，本书不写死，以官方文档为准。**

> **误解 vs 实际：** "Claude 和 ChatGPT 是同类东西，二选一就行。" —— 严格说，**Claude 既是模型系列、也常用来指它的产品；而 ChatGPT 是产品，它的内核模型叫 GPT**。所以"Claude vs ChatGPT"其实是拿"内核+外壳"去比"外壳"，有点错位。日常这么说没大问题，但心里要清楚那层关系。

## 三、第②层 产品：包给"人"用的那层

光有厨师（模型）你还吃不上饭，得有一家**餐厅**：菜单、服务员、餐桌、收银。**产品**就是这层包装——它把模型套上一个聊天界面、登录、历史记录、文件上传等等，让一个普通人**打开就能用**。

- ChatGPT、Claude 的 App 和网页、Gemini 的 App，都是产品。它们背后是各自的模型。
- 同一个内核，包装可以很不一样：网页版、手机 App、浏览器插件、集成在某个办公软件里……**对人友好**是产品这层的全部目的。

> **为什么要分清这一层？** 因为你日常用的、能直接点开聊天的，**几乎都是"产品"**。当有人问"哪个 AI 好用"，他其实在比的是产品体验（第18章会讲怎么选），而不是在比裸模型。把"产品"和"模型"分开，你才不会被名字绕进去。

## 四、第③层 API：开给"程序"用的接口

餐厅是给走进来的客人吃的。但如果**另一家奶茶店**想在自己的菜单里加一道"招牌咖喱饭"，它不必重新雇厨师——可以通过厨师的**接单窗口下单**：把订单发过去，厨师做好递出来，奶茶店拿去自己摆盘、卖给自己的客人。

**API（应用程序接口）**就是这个"接单窗口"，只不过下单的不是人，是**程序**。

- 你自己的网站、App、脚本，可以通过 API 把问题发给模型、把回答拿回来，**嵌进你自己的产品里**。
- 用 API 通常需要一把**密钥（API key）**来证明"是你在下单"，并据此**按 token 计费**（第3、38章）。
- 这把密钥很敏感——**泄露了等于别人能拿你的账号点单、记你的账**。所以"千万别把密钥贴进聊天框、别提交到公开仓库"是铁律（第37章）。

> **实际会怎么遇到：** 当你或你同事说"我们想在自己的系统里接入 AI"——比如让公司官网的客服自动回话——那要做的就是**调 API**，不是去开个 ChatGPT 会员。这正是第35章"给现有程序接上 AI"那个项目要动手做的事。

## 五、第④层 Agent：会"自己动手"的那层

到这里，前三层的厨师都还只是"你点什么、他做什么"。**Agent（智能体）**则更进一步：你只告诉他**目标**——"今晚搞一桌四菜一汤"——他自己去**列菜单、查库存、缺什么去采购、一道道做、摆盘、最后收拾厨房**。中途遇到问题（青菜不新鲜了），他会自己想办法（换一道）。

把这套搬到 AI 上，Agent 就是：**给模型配上"工具"（能读写文件、能运行命令、能联网、能调别的程序），再让它围绕一个目标，自己跑"调查→动手→检查→再调整"的循环**。

| | 聊天产品（第②层） | Agent（第④层） |
| --- | --- | --- |
| 它能做什么 | 用文字回答你 | 真的去**动手操作**（改文件、跑命令等） |
| 谁来执行 | 你照着它说的，自己去做 | 它自己执行，你在旁边把关 |
| 推进方式 | 你问一句它答一句 | 朝着目标自己往前走好几步 |

本书第四部分要学的 **Claude Code、Codex，就是专门干"编程"这件事的 Agent**——你说"给这个页面加个搜索功能"，它自己去读代码、改文件、跑测试（第23、24章）。

> **误解 vs 实际：** "Agent 不就是更聪明的聊天机器人吗？" —— 关键差别**不在'更聪明'，而在'能动手'**。也正因为它能真的操作你的东西，风险也升级了：它可能擅自做了你没料到的事。所以 Agent 一定配着"许可/确认"机制和"能回滚"的环境来用（第26、27、40章）。能力和风险，是同一枚硬币的两面。

## 六、四层是怎么叠起来的

把四层串起来看一个完整画面：

```text
        ┌─────────────────────── Agent（会自己动手跑流程）
        │   ┌─────────────────── 产品（给人用的界面）
        │   │   ┌─────────────── API（给程序用的接口）
        │   │   │   ┌─────────── 模型（会思考的内核）
   你 → 目标   点单   调用        预测下一个词
```

- 最里面永远是**模型**。
- 往外，要么包成**产品**给人用，要么开成 **API** 给程序用。
- 再往外，给它工具和目标，就成了 **Agent**——而 Agent 内部，往往又是通过 API 在调用模型。

所以"模型 / 产品 / API / Agent"不是让你四选一，而是帮你认清：**我现在面对的，是哪一层？**

## 七、实用判断：我面对的是哪一层？

下次再遇到一个 AI 名词或需求，问自己三个问题就能定位：

1. **是人直接在界面里用，还是程序在背后调用？** —— 人用 → 多半是"产品"；程序调 → "API"。
2. **它只会"说"，还是会"动手做"？** —— 只回答 → 产品/模型层；能改文件、跑命令、联网 → 带"Agent"性质。
3. **这个名字是公司的内核技术，还是一个能打开的应用？** —— 内核 → "模型"；能打开就用 → "产品"。

> **要点：** 当同事说"我们上 AI 吧"，最该先问的不是"买哪个会员"，而是 **"是人来对话框里用，还是要让我们的系统自动调用，还是要搭一个会自己跑的 Agent？"** 这三件事的成本、门槛、风险完全不同。先分清在哪一层，再谈选型——这能省掉大量来回。

## 八、常见误解，集中纠正

| 误解 | 实际 |
| --- | --- |
| Claude 和 ChatGPT 是同一种东西在比 | Claude 多指模型/产品，ChatGPT 是产品、内核叫 GPT，比的不是同一层 |
| 模型可以"下载下来直接聊" | 模型是内核，得装进产品或通过 API 才能用（本地模型是另一种情况，见第21章） |
| 接入 AI = 开个会员 | 让**程序**用 AI 要走 API；开会员是给**人**用产品 |
| Agent 就是聊天机器人升级版 | Agent 的本质是"能动手 + 自己跑循环"，风险也随之升级 |
| 选 AI 就是选最强的模型 | 普通人选的其实是"产品体验是否趁手"，强弱只是其中一项（第18章） |

## 小结

- AI 的名词大多能塞进**四层**：**模型**（会思考的内核）→ **产品**（给人用的界面）→ **API**（给程序用的接口）→ **Agent**（会自己动手跑流程）。
- 它们是**层层包裹**的关系，不是四选一：同一个模型，可包成产品、开成 API、做成 Agent。
- 日常你点开就聊的，几乎都是**产品**；让自己的系统接入 AI，走的是 **API**；会替你动手做事的，是 **Agent**。
- 判断"在哪一层"的关键问题：**人用还是程序用？只会说还是会动手？** 
- Agent 的能力和风险是一体的——它能动手，所以必须配许可与回滚机制（第四、六部分）。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。先自己想，再看引用块里的答案解析。

1. **[基础·概念题]** "Claude 是模型，ChatGPT 是产品"——这句话的意思最接近？
   - A. 两者没区别
   - B. 模型是内核（会算的那个东西），产品是包着内核、给你用的那层界面
   - C. Claude 更贵
   - D. ChatGPT 不用模型
   > **答案 B。** ChatGPT 是 OpenAI 包在 GPT 模型外面的产品；Claude 既是模型系列名、也常被用来指它的产品。记住"内核 vs 外壳"这层关系，名词就不乱了。

2. **[基础·概念题]** API 大致是什么？
   - A. 一个网页
   - B. 让你自己的程序去"调用"模型能力的接口
   - C. 一种付费会员
   - D. 一个聊天机器人的名字
   > **答案 B。** 网页/App 是给人用的；API 是给**程序**用的同一个能力入口。第35章"给程序接 AI"用的就是它。

3. **[基础·误区题]** "Agent 就是更聪明的聊天机器人"——错在哪？
   - A. 没错
   - B. Agent 的关键不在"更聪明"，而在它能**真的动手**（调工具、读写文件、跑多步），风险也由此而来
   - C. Agent 不需要模型
   - D. Agent 只能聊天
   > **答案 B。** 聊天机器人只会"说"；Agent 会"做"。能力升级的同时，"它擅自做了我没料到的事"这种风险也跟着来了（呼应第19、40章）。

4. **[进阶·情景题]** 同事说"我们接入 AI 吧"。要判断他到底想要哪一层，最该先问哪个问题？
   - A. "买哪个会员？"
   - B. "是人来对话框里用，还是要让我们自己的系统自动调用？"
   - C. "用不用换电脑？"
   - D. "AI 会不会取代我们？"
   > **答案 B。** "人用产品"、"程序调 API"、"搭一个会自己跑的 Agent"是三件不同的事，成本与风险都不同。先分清在哪一层，再谈选型。

5. **[基础·情景题]** 你只是想偶尔让 AI 帮你改改文章，下面哪种最划算省心？
   - A. 直接用现成的聊天产品
   - B. 自己写程序调 API
   - C. 在本地搭一个模型
   - D. 雇一个工程师
   > **答案 A。** 个人、低频、对话式需求，现成产品就是为你准备的。API/本地模型/Agent 是"产品满足不了"时才往下走的台阶，别一上来就上重的。

6. **[进阶·操作/观察题]** 挑一个你在用的 AI 工具，试着说出它处在哪一层：它背后大概是什么**模型**？你用的是**产品**界面还是 **API**？它能不能**动手**（联网、读文件、调工具）从而带点 **Agent** 味？
   > **该观察到：** 多数人日常用的是"产品"；当它能联网、能读你上传的文件、能调用工具时，就开始带 Agent 性质了。能对号入座，这张地图就装进脑子里了。
