你平时用的 AI，几乎都住在"别人家"——你打的字飞过网络，到某家公司的服务器上算完，再把答案传回来。这套很省心，但也带来三件让一部分人不舒服的事：你的内容得**离开你的电脑**才能被处理；**断了网就用不了**；用得多还得**按量掏钱**。

于是有人开始琢磨另一条路：能不能把模型**直接装进自己的电脑**，让它在本地跑，不联网、不外传、不计费？答案是能——这就是这一章要讲的**本地模型（local model）**，以及帮普通人把它跑起来的两类工具，Ollama 和 LM Studio。先说清楚一件事：本地模型不是"更高级的版本"，而是"换个地方跑的版本"，它有它的甜头，也有它实打实的代价。

## 一、为什么有人想在自己电脑上跑模型

把模型搬回家，图的就是三样东西。先看清楚它们到底解决了什么问题。

| 想要的 | 它解决了什么 | 谁会特别在意 |
| --- | --- | --- |
| **私密** | 内容全程不出本机，不上传到任何公司的服务器 | 处理敏感资料、不想数据"出门"的人 |
| **离线** | 没网也能用——飞机上、内网里、网断了照样跑 | 网络受限、或需要在隔离环境里干活的人 |
| **不按次付费** | 装好之后随便用，不再为每一次调用计费 | 用量很大、或不想被计费心理负担拖着的人 |

这三样的共同点是：它们都来自同一件事——**模型在你自己的机器上运行，输入输出都不离开本机**。云端那套之所以省心，恰恰是因为重活在"别人家"干；而本地这套之所以私密、离线、不计费，也正是因为重活搬回了"你家"。**好处和代价，是同一个原因的两面。**

> **要点：** 别把"本地"当成一种炫技，它本质上是一道**取舍题**。你用"自己机器的算力和一点配置麻烦"，换来"私密、离线、不计费"。想清楚你最在意哪一头，再决定走不走这条路——而不是看到"能装进自己电脑"就觉得高人一等。

## 二、本地 vs 云端：一张表把取舍摆平

这是全章最该记住的一张表。本地和云端不是"谁更好"，而是"为不同的人、不同的场景准备的"。

| | 云端（你平时用的那种） | 本地（装进自己电脑） |
| --- | --- | --- |
| 数据去哪 | 发到公司服务器再算 | **不出本机** |
| 要不要联网 | 要 | **不用** |
| 花钱方式 | 按量计费 / 套餐额度 | 装好后**不按次付费**（但你得有一台够格的电脑） |
| 能力强弱 | 通常**更强**（背后是巨型机器） | **受你的机器限**，同等条件下常**不如**云端顶配 |
| 上手难度 | 打开就用，省心 | 要下载、要配置，有一点门槛 |
| 维护 | 公司替你扛 | 你自己管 |

一句话概括两边的性格：

- **云端 = 强大、省心，但数据要出门、要按量花钱。**
- **本地 = 私密、离线、不计费，但受你的机器限、有配置门槛、通常不如云端强。**

注意最容易被忽略的一行——**能力强弱**。云端跑的往往是非常大的模型，背后是普通人买不起的算力；你电脑上能跑的，多半是它的"瘦身版"。所以指望"装个本地模型，效果和最强的云端一样"，多半要失望。这不是 Ollama 或 LM Studio 不行，是你那台电脑的"体格"决定了它跑得动多大的模型。

> **要点：** 选本地还是云端，先问自己一句话：**"这件事，最要紧的是数据别出门，还是效果要最好？"** 要数据别出门、要能离线、要不计费——本地有它不可替代的价值；要顶配效果、要省心——老老实实用云端。这跟第17章那张"模型/产品/API/Agent"地图是一个道理：先看清需求在哪一层，再谈用什么。

## 三、下载前先看一眼：你的电脑扛不扛得住

这是本地模型最实际、也最容易被新手忽略的一道坎。**不是所有模型你的电脑都跑得动。**

打个比方：模型就像一件"行李"，你的电脑就像一辆车。行李有大有小，车的后备箱也有大有小——**箱子塞不下，车就开不动**。模型这件"行李"的大小，主要看两个东西能不能装下它：

- **内存（RAM）**：电脑临时干活的空间。模型跑起来要整个"摊开"在内存里，模型越大，吃的内存越多。
- **显存（VRAM）**：显卡（GPU）上的专用内存。很多模型用显卡跑会快很多，但显卡的"专用空间"通常比内存还紧张。

落到实处它实际是什么意思：每一个能下载的本地模型，发布方一般都会标出它**大致需要多少内存/显存**。下载之前，拿这个要求去对照你电脑的配置——**装得下，它才跑得顺；硬塞一个超出配置的大模型，轻则慢得令人发指（卡成幻灯片），重则根本跑不起来。**

```text
下载前的"对一眼"动作：

  模型标的要求：       你的电脑：           结论
  ─────────────       ─────────           ──────
  需要 16 GB 内存  vs  你有 8 GB     →  跑不动 / 极慢
  需要 8 GB 内存   vs  你有 16 GB    →  没问题
  （具体数字以模型发布方/官方文档为准，本书不写死）
```

怎么知道一个模型要多大、自己电脑又有多少？这些**具体数字和查看方式各家不同、且常更新，以官方文档为准**。你只要先建立这个意识：**本地模型是有"硬件门槛"的，下载前先对一眼配置**，别下完了才发现跑不动，白等半天。

> **要点：** "能不能跑本地模型"不取决于你想不想，而取决于你的**机器扛不扛得住**。养成一个习惯：看到一个本地模型，先找它标注的内存/显存要求，再看自己电脑的配置——这一步只花一分钟，能省掉"下载几个 GB、跑起来却卡死"的折腾。

## 四、量化：把大模型"压瘦"一点好上车

你可能会问：那大模型就跟普通电脑彻底无缘了吗？也不全是。这里有个常听到的词——**量化（quantization）**，值得用一句话搞懂。

接着上面"行李"的比喻：量化就像把行李里的东西**用真空压缩袋压一压**，体积变小了，更容易塞进小后备箱。代价是——压得太狠，衣服可能会有点皱。

落回它实际是什么：量化是一种**把模型"压缩"得更小、更省内存**的技术，让原本跑不动的模型，在普通电脑上也有机会跑起来。代价是**压得越狠，效果通常会掉一点**（回答质量可能略降）。所以你常会看到同一个模型有好几个"压缩档位"可选——压得轻，效果好但更吃配置；压得重，省资源但质量打点折。

> **要点：** 量化是本地模型世界里的常见取舍：**用一点"质量"换"能在你的电脑上跑得动"**。普通人不用懂它怎么实现，只要知道——同一个模型给你好几个版本挑时，那多半是不同的量化档位，**在"跑得动"和"答得好"之间选一个平衡点**。具体怎么选、各档差多少，以官方文档为准。

## 五、Ollama 与 LM Studio：帮普通人把模型跑起来的工具

讲了半天"本地模型"，可模型本身只是个内核（还记得第17章吗——模型是"会思考的脑子"，得有东西把它包起来才好用）。**直接和裸模型打交道是很折腾的**：要下载、要配置、要让它跑起来、还要有个地方跟它对话。

**Ollama 和 LM Studio，就是替你把这些麻烦事包办了的工具**——它们让普通人能比较省心地在本地**下载**模型、**运行**模型、并**跟它对话**。

这里顺带补一个常见词：你能这样自由下载、装进自己电脑跑的模型，多半属于**开放权重模型（open-weight model）**——简单说，就是发布方把模型"放出来"、允许大家下载到本地使用的那一类。正因为有这类模型可下载，本地这条路才走得通；而很多最顶尖的云端模型并不开放下载，只能通过它们的产品或 API 用（第17章）。哪些模型是开放权重、能不能商用，各家条款不同，**以官方文档为准**。

- 一个大致的印象（**具体功能、界面、支持的模型以官方文档为准**）：它们通常能帮你浏览和下载可用的本地模型、管理你下载下来的几个模型、并提供一个能直接聊天的入口。
- 两者定位略有不同、各有侧重，但对普通人来说，目的是一样的：**把"在本地跑模型"这件原本挺技术的事，变得点几下就能开始。**

至于具体怎么装、怎么下模型、命令和按钮长什么样——这些**随版本变动频繁，本书不写死，请以各自官方文档为准**;真要动手的深度操作，留到附录 C 再细讲。

> **要点：** 别把 Ollama / LM Studio 和"模型"本身搞混。它们**不是模型**，而是帮你把模型**装下来、跑起来、用起来**的"管家"——就像餐厅之于厨师（第17章）。换个管家，跑的还是同一批本地模型；真正决定效果的，是你跑的是**哪个模型**、你的**电脑多强**，而不是用哪个管家。

## 六、常见误解，集中纠正

| 误解 | 实际 |
| --- | --- |
| 本地模型一定比云端更强 | 恰恰相反——本地受你机器的算力限，同等条件下**常不如**云端顶配；强弱看**具体型号 + 你的硬件**，不看"本地"二字 |
| 本地模型 = 绝对安全 | 它更**私密**（数据不出本机）这点没错，但"私密"不等于"安全"——模型照样会幻觉、会出错（第9章），下载来路不明的模型也有风险 |
| 装了 Ollama 就有了"一个更好的 AI" | Ollama/LM Studio 只是**运行工具**，不是模型本身；真正干活的是你下载的那个模型 |
| 随便挑个模型下下来就能跑 | 先看它的**内存/显存要求**对不对得上你的电脑，否则会极慢甚至跑不起来 |
| 本地就是给极客玩的，跟我没关系 | 如果你**特别在意数据不外传、要离线用、或用量很大**，本地是有现实价值的一条路——只是要一点配置门槛 |

## 小结

- 有人选**本地模型**，图的是三样：**私密**（数据不出本机）、**离线**（没网也能用）、**不按次付费**——它们都源于"模型在你自己机器上跑"。
- 本地 vs 云端是一道**取舍题**：云端**强大省心但数据出门、按量计费**；本地**私密离线不计费，但受机器限、有门槛、通常不如云端强**。
- 下载前**先看硬件要求**：模型要"摊"进你的**内存/显存**，装不下就跑不动甚至跑不起来（具体数字以官方文档为准）。
- **量化**是用一点质量换"在普通电脑上跑得动"的常见取舍；同一模型的多个版本多半是不同量化档位。
- **Ollama / LM Studio** 是帮普通人在本地下载、运行、对话的**工具**，不是模型本身；具体用法以官方文档为准，深度操作见附录 C。

下一章我们换个角度：模型再强，光会"说"还不够——怎么给它接上**外部的工具和数据**，让它能读你指定的资料、按你的固定流程办事？这就要讲 Skills 和 MCP 了。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。答案与解析在每题下方的引用块里——先自己想，再对照。

1. **[基础·概念题]** 人们想在自己电脑上跑本地模型，主要图的是下面哪三样？
   - A. 私密、离线、不按次付费
   - B. 一定更强、一定更快、一定更便宜买电脑
   - C. 自动联网、自动更新、自动备份
   - D. 永远不会出错、永远不会幻觉
   > **答案 A。** 本地模型的三大甜头——数据不出本机（私密）、没网也能用（离线）、装好后不计费——都来自"模型在你自己机器上跑"这一件事。B 把本地误当成"更强更快"（实际常不如云端顶配）；D 把"私密"错当成"不会出错"，模型照样会幻觉（第9章）。

2. **[基础·概念题]** 关于"本地"和"云端"，下面哪句最准确？
   - A. 本地永远比云端好
   - B. 云端强大省心但数据要出门、按量计费；本地私密离线不计费但受机器限、通常不如云端强
   - C. 云端不用联网，本地需要联网
   - D. 两者完全一样，只是名字不同
   > **答案 B。** 这是一道取舍题，没有"谁永远更好"。A 是最常见的误解；C 把两者的联网关系**说反了**（云端才要联网，本地不用）；D 忽视了数据去向、计费、能力等一连串实打实的区别。

3. **[基础·误区题]** "本地模型一定比云端的更强、更聪明。"——错在哪？
   - A. 没错，本地就是更强
   - B. 本地受你电脑算力限制，同等条件下常不如云端顶配；强弱看具体型号和你的硬件，不看"本地"二字
   - C. 错在本地其实更弱，永远不能用
   - D. 错在本地和云端跑的是完全不同的东西
   > **答案 B。** 云端背后是普通人买不起的巨型算力，你电脑能跑的往往是"瘦身版"。决定强弱的是**哪个模型 + 你的硬件**，"本地"这两个字本身不代表强。犯这个错的人，通常是把"装进自己电脑"误读成了"更高级"。C 走到了另一个极端（本地也有它不可替代的价值）。

4. **[进阶·误区题]** "本地模型数据不出门，所以它绝对安全、不会出错。"——哪里不对？
   - A. 完全正确
   - B. "更私密"≠"绝对安全"：模型照样会幻觉、会出错，下载来路不明的模型本身也有风险
   - C. 本地模型其实会偷偷上传数据
   - D. 本地模型没有任何隐私好处
   > **答案 B。** "数据不出本机"确实让它更**私密**，但"私密"和"安全/正确"是两回事——它的回答仍可能是错的（第9章幻觉照样存在），而且从不可靠的来源下模型本身就有风险。把"私密"等同于"安全"，是这道题想纠正的混淆。D 又否认了它真实存在的隐私好处，同样不对。

5. **[基础·情景题]** 你看中一个本地模型，下载之前**最该先做**的一件事是什么？
   - A. 直接点下载，跑不动再说
   - B. 看一眼它标注的内存/显存要求，对照自己电脑的配置扛不扛得住
   - C. 先把云端会员退掉
   - D. 把电脑里其他软件全部卸载
   > **答案 B。** 模型要整个"摊"进你的内存/显存，**装不下就会极慢甚至跑不起来**。花一分钟对一眼配置，能省掉"下载几个 GB 却跑成幻灯片"的折腾（具体数字以官方文档为准）。A 是最容易踩的坑；C、D 都没对准"硬件够不够"这个真正的关口。

6. **[基础·操作/观察题]** 先查一下你自己电脑的内存（RAM）大概有多少 GB;再去任意一个本地模型工具（如 Ollama / LM Studio，**具体以官方文档为准**）看看，同一个模型常常有好几个不同大小的版本。观察：那些更小的版本，是不是更省资源、但可能效果差一点？
   > **该观察到：** 同一模型的多个版本，多半是不同的**量化（quantization）**档位——压得越狠、文件越小、越省内存，但回答质量可能略降；压得越轻、越吃配置、效果越好。这正是本地模型的核心取舍：**在"跑得动"和"答得好"之间挑平衡点**。亲手对一次自己的内存和模型大小，"我这台电脑能跑多大的模型"这个直觉就建立起来了（各档差多少以官方文档为准）。
