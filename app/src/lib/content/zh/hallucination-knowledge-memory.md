它能写诗、能改代码、能把一篇长文总结得头头是道，可你也一定撞见过两件扫兴的事：它**一本正经地说错**——引了一篇根本不存在的论文、报了个查无此人的"专家"、把数字编得有鼻子有眼；以及它**记不住你**——昨天才告诉过它的事，今天新开一个对话，它一脸茫然。

这两件事不是它"不靠谱"或"偷懒"，而是它的工作方式带来的必然结果。这一章把它们一次讲透：**幻觉**（为什么会说错）、以及**知识 vs 记忆**（什么记得牢、什么转头就忘）。搞懂这一章，你对它"该信几分、东西该往哪放"就有谱了——这也是全书最关乎"别踩坑"的一章。

## 一、幻觉：把不存在的东西，说得特别自信

**幻觉（hallucination）**，指的是模型把**不存在的或错误的**内容，说得**有模有样、信誓旦旦**。注意重点不在"它错了"——谁都会错——而在"它**错得毫无破绽、还特别自信**"。

为什么会这样？答案在第4章那句话里：

> **它追求的是"像真的"，不是"是真的"。** 模型的本职是根据前文，挑出"接下去最像样、最顺的词"（第4章），它并没有一个"事实数据库"可以去核对真假。所以当它"不知道"时，它不会停下来说"我没把握"——它会顺着语言的惯性，**编一个读起来最合理的答案**。一篇假论文之所以编得像真的（像样的标题、像样的作者、像样的期刊），正因为它见过成千上万篇真论文长什么样，于是仿着那个样子**造**了一篇。它不是在撒谎（撒谎得先知道真相），它是在"**把话顺下去**"，顺着顺着就顺出了假的。

理解了成因，对策就很自然——三招，从弱到强：

| 对策 | 怎么做 | 管什么用 |
| --- | --- | --- |
| **自己复核** | 关键事实（数字、引用、人名、日期、法律医疗结论）一律自己再查一遍 | 兜底，最重要的一招 |
| **让它给依据** | 要求它"列出处/来源""说说你凭什么这么讲" | 没依据或依据可疑，就是警报 |
| **给它材料** | 把可靠资料贴进去，让它"**只**基于这些来答" | 从源头压住它瞎编的空间 |

> **要点：** 一个最反直觉、却最该刻进脑子里的判断——**它说得越流畅、越自信，越不代表越可靠**。流畅自信只说明它"接得顺"，跟"是不是真的"毫无关系。真正的安全习惯是反过来的：**越是要紧、越是它张口就来的事实，越要亲手验。** 第三招"给它材料、让它基于材料答"，正是后面 RAG（第20章）那一整套技术的核心思路——与其赌它记得对，不如把对的资料喂到它面前。

## 二、知识 vs 记忆：一张表分清"记得牢"和"转头忘"

它脑子里的东西，其实分**两种**，脾气完全不同。混淆这两种，是初学者最常栽的地方。

- **知识（knowledge）**：训练时**焊进参数里**的常识和事实。它张口就来，不依赖你这次给的材料——你问"水的沸点""唐朝在宋朝前还是后"，它直接答得上。
- **记忆（memory）**：只活在**这次对话**里的临时信息，全靠上下文窗口（第3章那张"桌面"）撑着。你这次告诉它你叫什么、什么口味——这些是"记忆"，**关掉对话或被挤出窗口，就没了**。

并排看：

| | 知识（knowledge） | 记忆（memory） |
| --- | --- | --- |
| 存在哪 | **焊进参数里**（训练时学的） | 只在**这次对话的上下文**里 |
| 哪来的 | 海量训练文本 | 你这次对话喂进去的 |
| 能撑多久 | 长期都在（但有"截止日期"，见下） | 关掉/超出窗口就忘 |
| 张口就来吗 | 是，不靠你给材料 | 不是，全靠这次摊在桌面上 |
| 典型例子 | "巴黎是法国首都" | "我刚说过我对花生过敏" |

知识有一个绕不开的硬限制——**知识截止（knowledge cutoff）**：模型的训练数据有个"最后期限"，**这个时间点之后**才发生的事、才出的新版本、才冒出的新名词，它**本身并不知道**，硬问它，它很可能直接给你**幻觉**一个出来。

> **要点：** 各模型的知识截止时间不同、且会随版本更新，本书不写死，**以官方文档为准**。你只需记住直觉：**问"最近发生了什么"，默认别信它张口就来的版本**——要么让它联网查（第19章，查到也要复核），要么把最新资料贴给它读。它的"知识"是某一刻的快照，不会自动更新。

## 三、最该破除的误解：它像人一样长期记得你吗？

这是本章——也是全书——最要紧、最该掰扯清楚的一点。接着第3章那句话往深里讲。

你和它聊了很久，难免觉得它"认识"你了。但请记住第3章的结论：**模型本体没有跨对话的记忆**。新开一个对话，桌面是**全空的**，它对你一无所知；哪怕同一个对话聊太长，最早说的内容也会被挤出窗口而"忘掉"。它不是"记性差"，而是**本体压根没有一个能跨对话存东西的地方**。

可你一定会立刻反问：**那 Claude 的 Projects、ChatGPT 的 Memory（记忆）又是怎么回事？我明明见过它"记住"我的偏好、记住整个项目啊！** 这是必须讲清楚的关键——

> **模型本体确实没有跨对话记忆；但很多产品会在模型之外的"产品层"额外提供"记忆"能力**：比如长期记忆、项目记忆、项目上下文文件、账号级偏好、企业知识库等（你会遇到的有 Claude Projects、ChatGPT 的 Memory、IDE 里的项目上下文文件如 CLAUDE.md、账号里的个性化设置、公司内部知识库）。但这**不是模型本体长出了记忆，而是产品层每次自动帮你把相关信息重新塞回这次对话的上下文里**。所以你看到这些功能并不矛盾——掀开盖子，底下还是"无记忆的模型 + 每次重新喂上下文"（正是第3章那张"桌面"）。各家具体怎么记、记多久、默认开还是关、能不能删，**以官方文档为准**。

把这两层分清楚，胜过记住任何一个产品的用法：

| | 模型本体 | 产品层的"记忆" |
| --- | --- | --- |
| 是什么 | 那个只会按概率接话的内核 | 产品在模型外面加的一层功能 |
| 记不记得你 | **不记得**，每次空白开始 | 帮你把"该记的"存起来 |
| 怎么做到的 | —— | **每次对话时，自动把这些信息重新塞回这次的上下文**（又回到那张桌子） |
| 它"记得"的边界 | —— | 永远只是**产品决定塞回桌面的那部分**；没塞回来的，它照样不知道 |

> **要点：** 理解了"**底层无记忆、上层靠重新喂**"，你就能预判这些功能的脾气：它能"记得"的，永远是产品替你塞回上下文的那部分；你以为它该记得、但产品没塞回来的，它就是不知道。这也解释了为什么有时它"莫名其妙忘了"你设过的偏好——多半是那条信息这次没被放回桌面。所以这不是 bug，是"产品层记忆"的工作方式。

## 四、把前面的"误解"收个尾

本章有两个相互呼应的误解，并排放一起最清楚：

- **"它说得越自信越可靠"** —— 错。自信只代表"接得顺"，和真假无关；越要紧、越张口就来的，越要自己验。
- **"它像人一样长期记得我"** —— 错。模型本体没有跨对话记忆；你看到的"记得"是产品层每次把信息重新喂回上下文，不是它本人记住了。

这两件事其实是**同一个底层事实的两面**：它没有一个"真相数据库"、也没有一个"长期记忆库"，它有的只是训练时压进参数的知识（有截止日期）+ 这次摊在桌面上的上下文。把这条记牢，它的"会说错"和"记不住"就都不再奇怪了。

## 五、常见误解，集中纠正

| 误解 | 实际 |
| --- | --- |
| 它说得越流畅、越自信，就越可靠 | 流畅自信只代表"接得顺"，和真假无关；越要紧的事实越要自己复核 |
| 它"说错"是偶尔走神，提醒一下就好 | 幻觉源于它追求"像真的"而非"是真的"，是本质特性；要靠复核、给依据、给材料来防 |
| 它知道最新发生的事 | 知识有**截止日期**，之后的事它本身不知道，硬问会幻觉；要联网或喂资料（以官方文档为准） |
| 你这次告诉它的事，它会一直记得 | 那是"记忆"，只活在这次对话里，关掉或被挤出窗口就忘 |
| 有"记忆"功能，说明模型本体记住了我 | 是**产品层**每次把信息重新塞回上下文，不是模型本体记得；具体以官方文档为准 |

## 小结

- **幻觉（hallucination）**：把不存在/错误的东西说得很自信。成因是它追求"**像真的**"不是"**是真的**"（第4章）。对策：关键事实自己复核、让它给依据、给它材料让它"只基于材料答"（通向第20章 RAG）。
- 越流畅自信 ≠ 越可靠——这是最该刻进脑子的反直觉判断。
- **知识 vs 记忆**：知识是**焊进参数**的、张口就来，但有**知识截止（knowledge cutoff）**；记忆只活在**这次对话**里，关掉就忘。
- 模型本体**没有跨对话记忆**；你看到的 Claude Projects、ChatGPT Memory、项目上下文文件、账号偏好、企业知识库等，是**产品层每次把信息重新塞回上下文**，不是模型本体记得——以官方文档为准。
- 一句话收束：它既没有"真相库"也没有"长期记忆库"，只有**参数里的知识（有截止）+ 这次桌面上的上下文**。它会说错、会忘，皆源于此。

第一部分到这里就读完了——你已经把模型"是什么、怎么运转"的直觉建立起来了。从下一部分起，我们转向**怎么把它用好**：从第一次认真对话，到把"我想要什么"说清楚的提示词功夫。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。答案与解析在每题下方的引用块里——先自己想，再对照。

1. **[基础·概念题]** "幻觉（hallucination）"指的是？
   - A. 模型死机、卡住不动
   - B. 模型把不存在或错误的内容，说得有模有样、很自信
   - C. 模型把你的对话泄露出去
   - D. 模型回答得太慢
   > **答案 B。** 幻觉的要害是"错得毫无破绽、还很自信"——成因是它追求"像真的"而非"是真的"，没有事实库可核对，不知道时就顺着语言惯性编一个。A、D 是性能问题，C 是隐私问题（第37章），都不是幻觉。

2. **[基础·概念题]** 下面哪一对，正确区分了"知识"和"记忆"？
   - A. 知识只活在这次对话里，记忆焊在参数里
   - B. 知识是焊进参数、张口就来的（但有截止日期）；记忆只活在这次对话里，关掉就忘
   - C. 两者是一回事
   - D. 知识和记忆都会自动更新到最新
   > **答案 B。** 知识来自训练、长期都在但有"知识截止"；记忆靠这次的上下文窗口撑着，关掉或被挤出就没了。A 把两者说反了；D 错在知识不会自动更新（有截止日期）、记忆更不会跨对话留存。

3. **[进阶·误区题]** 它给你一段回答，引经据典、口吻笃定。有人说"讲得这么顺这么有把握，肯定靠谱"。错在哪？
   - A. 没错，自信就代表可靠
   - B. 流畅自信只说明它"接得顺"，和真假无关；越要紧、越张口就来的事实，越要自己复核
   - C. 它笃定是因为联网查过了
   - D. 它引用了来源就一定为真
   > **答案 B。** 这是最该破除的直觉——它越顺越要警惕，因为"顺"正是幻觉编得像真的的表现。C 不能假设（默认它没联网）；D 也危险——它连"来源"都可能是编的，所以要让它给依据**并自己去核**。

4. **[进阶·误区题]** "我用的产品有'记忆'功能，能记住我的偏好，所以模型本体是记得我的。"哪里不对？
   - A. 完全正确，模型记住了你
   - B. 那是**产品层**每次自动把相关信息重新塞回这次对话的上下文，不是模型本体长出了记忆
   - C. 说明这个模型比别的模型高级
   - D. 说明它把你的资料永久存进了参数
   > **答案 B。** 关键是分清两层：模型本体没有跨对话记忆，"记忆"功能是产品层每次帮你把信息重新喂回上下文（第3章那张桌子）。它能记得的，永远只是产品决定塞回来的那部分；具体怎么记、能否删，**以官方文档为准**。D 更是误解——产品层记忆不会改动模型参数。

5. **[基础·情景题]** 你想让它帮你了解"上个月刚发布的某新产品"的细节，它却答得头头是道。最稳妥的做法是？
   - A. 直接采信，它说得很详细
   - B. 警惕它可能在"知识截止"之后凭空编（幻觉）；让它联网查，或把官方资料贴给它，并自己核对关键点
   - C. 多问它几遍，答案一致就一定对
   - D. 让它把温度调高再答一次
   > **答案 B。** "最近发生的事"正是知识截止的盲区，它很可能编得有鼻子有眼。要么联网（第19章，查到也要复核）、要么喂它最新资料让它基于材料答。A 正中幻觉的陷阱；C 没用（编的东西也能反复编一致）；D 风马牛不相及，温度高只会更可能飘（第8章）。

6. **[基础·操作/观察题]** 故意问它一个**偏冷门又具体**的事实（如某本小众书的出版年、某条法规的具体条款），要求它"给出处"；再把一份你手上的可靠材料贴进去，让它"只根据这份材料"回答同一问题。对比两次。
   > **该观察到：** 不给材料时，它可能编出像模像样、却查无实据的"出处"——这就是幻觉，也印证了"越自信≠越可靠"；给了材料、要求它只基于材料答之后，准确度通常明显提升。这正是 RAG（第20章）的核心思路：与其赌它记得对，不如把对的资料喂到它面前。亲手对比一次，你就再不会盲信它张口就来的"事实"了。
