到现在为止，你认识的 AI，本质上是个"**会答问题的家伙**"：你问一句，它答一句；它给你建议、给你草稿、给你思路，但**真正动手去做的，始终是你**。它说"你应该把这个文件改一下"，文件还是得你自己去改。

这一章要讲的 **Agent（智能体）**，是一个**质的不同**：它不只是"告诉你怎么做"，而是**自己动手去做**——自己读文件、自己跑命令、自己联网、自己调用别的程序，围着一个目标一步步往前推。第17章那张"四层地图"的最外层就是它；这一章把它讲透，也为第四部分要学的"编程智能体"（第23、24章）打好底子。

记住一句话，整章都在解释它：**Agent 的全部能力和全部风险，都来自同一件事——它能"真的动手"。**

## 一、从"答问题的 AI"到"会动手的 AI"

先看清楚这两类 AI 的分界线。

| | 聊天式 AI（你熟悉的） | Agent（这一章） |
| --- | --- | --- |
| 它给你什么 | 一段**回答**（建议、草稿、思路） | 一个**结果**（事情真的被做了） |
| 谁去执行 | **你**照着它说的，自己动手 | **它自己**执行，你在旁边把关 |
| 怎么推进 | 你问一句、它答一句 | 朝着目标，自己往前走好几步 |
| 打个比方 | 一位**顾问**：只出主意 | 一位**助理**：直接帮你把事办了 |

用上一章那个厨师比喻接着说（第17章）：聊天式 AI 像一个只会"口述菜谱"的厨师——他告诉你怎么做菜，但锅还是你来掌；Agent 则是一个**真的下厨**的厨师——你说"今晚来顿家常菜"，他自己去看冰箱、列菜单、缺料去补、一道道炒出来、最后还收拾灶台。

> **要点：** 别把 Agent 理解成"更聪明的聊天机器人"。关键差别**不在"更聪明"，而在"会动手"**。一个只会答题、另一个会真的去操作你的东西（改文件、跑命令、联网下单……）——这是能力层级的跳变，不是同一个东西变强一点。

那它"动手"的能力从哪来？答案是上一部分埋下的伏笔——**工具**。

## 二、关键零件：工具调用（tool use）

模型本身只会做一件事：**根据前文，生成文字**（第4章）。它没有手、没有眼睛，本身碰不到你的文件，也上不了网。那 Agent 是怎么"动手"的？

靠的是给模型配上一套**工具（tool）**，并让它学会**工具调用（tool use）**——也就是：模型在生成文字的过程中，可以**"请求"去用某个工具**（"帮我读一下这个文件""帮我搜一下这个""帮我运行这条命令"），外面的程序替它真的执行，再把**执行结果**喂回给模型，让它接着往下想。

```text
没有工具的模型：
   你问 → 模型只能用"它脑子里的东西"答 → 一段文字（碰不到真实世界）

装上工具的 Agent：
   目标 → 模型说"我需要先读这个文件" → 程序真去读 → 把内容喂回给它
        → 模型看完说"再搜一下这个" → 程序真去搜 → 把结果喂回去
        → ……如此往复，直到它认为目标达成
```

- 工具可以是各种各样的：读写文件、运行命令、联网搜索、查数据库、调用某个外部服务……（具体某个 Agent 配了哪些工具、怎么配，**以官方文档为准**。给 AI 接工具这件事，第22章和第六部分还会专门讲。）
- 注意这里的分工：**"想用哪个工具、用它来干嘛"是模型决定的；"真的去执行"是外面的程序做的。** 模型负责动脑和发指令，程序负责动手——两者合起来，才有了"会动手的 AI"。

> **要点：** "Agent = 模型 + 工具 + 一个目标"。模型还是那个只会生成文字的内核，但**当它生成的文字可以变成"去做某件事"的指令、并被真的执行时**，它就从"只会说"变成了"会做"。工具调用，就是接在模型和真实世界之间的那双"手"。

## 三、Agent 的招牌动作：自己跑"调查 → 动手 → 检查"的循环

光能调一次工具还不算 Agent 的精髓。真正让它像"会做事"的，是它能**自己跑一个循环**，多步推进，不用你一步步喂。

这个循环大致是这样：

```text
        ┌─────────────────────────────────────────┐
        │  ① 调查：看看现状、收集需要的信息          │
        │  ② 动手：基于了解到的，执行一步操作        │
        │  ③ 检查：看结果对不对、离目标近了没        │
        │  ④ 调整：不对就改方法，再回到 ①            │
        └───────────────┬─────────────────────────┘
                        │ 直到它判断"目标达成"
                        ▼
                     交还给你验收
```

举个不涉及编程的例子体会一下：你让一个 Agent"帮我把这个文件夹里所有图片整理好、按日期分类"。它可能会：先**看**文件夹里有什么（调查）→ **读**每张图的信息、**建**好分类文件夹、**移动**文件（动手）→ **核对**有没有漏掉、有没有放错（检查）→ 发现某张没有日期信息，**换个办法**处理（调整）……一轮轮下来，直到整理完。整个过程**它自己往前推**，你只在关键处把关。

这正是第17章那句话的展开：**给模型配上工具，再让它围绕目标自己跑"调查→动手→检查→再调整"的循环。** 这个"自己跑循环"的能力，是它区别于聊天机器人的核心。

> **要点：** 聊天式 AI 是"**你推一下，它走一步**"；Agent 是"**给个目标，它自己走一段**"。能自主跑这个循环，意味着它能完成"需要好几步、中间还要根据结果随机应变"的任务——这是它真正有用的地方，也正是下一节要说的、它真正危险的地方。

## 四、能力和风险，是同一枚硬币

这是全章最该记牢的一点：**Agent 强在哪，就危险在哪——因为它们是同一件事。**

聊天式 AI 哪怕答错了，伤害也有限：它只是**说**了句错话，要不要照做、怎么做，**关都在你这儿**。但 Agent 不一样——**它会真的去操作你的东西**。于是：

| 这件事 | 让它**有用** | 同时让它**有风险** |
| --- | --- | --- |
| 它能动手改你的文件 | 帮你真的把活干了 | 也可能**改错、删错**了你不想动的东西 |
| 它能自己跑命令 | 自动化掉繁琐步骤 | 也可能跑出**你没料到的后果** |
| 它能自己决定下一步 | 不用你一步步盯 | 它的判断可能**出错**，还自顾自走下去 |
| 它能联网、调外部服务 | 用上实时信息和外部能力 | 也可能触发你**不想要的对外动作** |

说白了：**"它能替你做事"和"它能替你闯祸"，是同一个能力的两面。** 你给了它"手"，它既能用这双手帮你，也能用这双手出岔子——而且它出岔子时，往往**不会停下来问你**，会顺着自己的判断一路做下去。

> **要点：** 第17章那句话在这里落地了——**Agent 的能力和风险是一体的**。所以 Agent 绝不能"撒手不管"地用，它必须配两样东西：**①许可/确认机制**（关键动作要它先问过你、你点头才做），**②可回滚的环境**（万一做错了，能退回去）。这两样不是给你添麻烦，而是**让"会动手的 AI"能安全使用的前提**。具体怎么设许可、怎么回滚，第四部分和第六部分会专门讲（第26、27、40章）；本书也反复强调：**当它在你本地的东西上动手时，务必先用 Git 起步、先切分支**（第24章），好随时退回。

这也顺带说明了为什么本书把"安全使用"放在很重的位置：你越是要享受 Agent"会动手"的好处，就越要把"它动手"这件事管在可控范围里。

## 五、回扣第17章：Agent 在"四层地图"的什么位置

把这一章接回上一部分那张地图（第17章），就一目了然了：

```text
   ┌──────────────────────── Agent（会自己动手跑流程）← 本章
   │   ┌──────────────────── 产品（给人用的界面）
   │   │   ┌──────────────── API（给程序用的接口）
   │   │   │   ┌──────────── 模型（会思考的内核）
你→目标   点单   调用          生成下一段文字
```

- 最里面永远是那个**只会生成文字**的模型（第4章）。
- Agent 是在它外面**套上工具、加上一个目标、让它自己跑循环**而成的最外层。
- 而 Agent 内部"动脑"的那部分，往往又是通过 **API**（第17章）在调用模型——所以这几层不是互斥的，而是层层包裹。

> **要点：** 当你遇到一个 AI、想判断它"是不是 Agent"，问一个问题就够了：**它只会"说"，还是会"真的动手做"（改文件、跑命令、联网、调工具）？** 只会说 → 还在产品/模型层；会动手、还能围着目标自己往前走 → 它就带上了 Agent 的性质。这正是第17章教你的"我面对的是哪一层"的判断法。

## 六、常见误解，集中纠正

| 误解 | 实际 |
| --- | --- |
| Agent 就是"更聪明的聊天机器人" | 关键差别不在"更聪明"，而在**会动手**：聊天 AI 只会说，Agent 会真的操作你的东西并自己跑循环 |
| Agent 凭空就能"动手" | 它靠**工具调用（tool use）**：模型发出"用某工具"的请求，外面的程序真去执行，再把结果喂回——模型动脑、程序动手 |
| 给个目标，撒手让它自己干就行 | Agent 会真的操作、还可能不停下来问你，必须配**许可/确认 + 可回滚环境**才能安全用（第26、27、40章） |
| 它能力强，所以更值得放心 | 能力和风险是**同一枚硬币**：能替你做事 = 能替你闯祸，越能干越要管好"它动手"这件事 |
| Agent 和模型/产品/API 是四选一 | 它们**层层包裹**：Agent = 模型 + 工具 + 目标，内部常通过 API 调模型（第17章） |
| 工具是模型自己执行的 | "用哪个工具"由模型决定，"真的执行"由外面的程序完成；分工不同 |

## 小结

- **Agent（智能体）**是会"**真的动手**"的 AI：不只告诉你怎么做，而是自己读文件、跑命令、联网、调程序，围着目标把事做完。
- 它动手的本事来自**工具调用（tool use）**：模型只会生成文字，但当它生成的文字变成"用某工具"的请求、被外面的程序真的执行时，它就从"会说"变成了"会做"。
- 它的招牌动作是**自己跑"调查→动手→检查→调整"的循环**——给个目标，它自己往前走一段，而不是你一步步喂（呼应第17章四层）。
- **能力和风险是同一枚硬币**：能替你做事，就能替你闯祸；它出错时还常常不停下来问你。所以必须配**许可/确认机制 + 可回滚环境**（第26、27、40章；本地动手务必先用 Git，第24章）。
- 在第17章那张地图上，Agent 是**最外层**：模型 + 工具 + 目标，内部常通过 API 调模型；判断"是不是 Agent"就看它**会不会真的动手**。

下一章我们换个方向，去看另一种"给 AI 加外挂"的常见办法——**RAG**：怎么给 AI 外挂一个资料库，让它答题前先"翻书"。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。答案与解析在每题下方的引用块里——先自己想，再对照。

1. **[基础·概念题]** 一个 AI 是"聊天式 AI"还是"Agent（智能体）"，最关键的分界线是？
   - A. 哪个用的模型参数更多
   - B. 它只会"说"（给你回答），还是会"真的动手做"（改文件、跑命令、联网、调工具）并围着目标自己往前推
   - C. 哪个回答得更快
   - D. 哪个界面更好看
   > **答案 B。** Agent 的要害不是"更聪明"，而是"会动手"——它能真的操作你的东西，还能自己跑多步循环。A、C、D 说的都是无关或次要的因素；把"参数多/更聪明"当成 Agent 的定义，正是最常见的误区。

2. **[基础·概念题]** Agent 之所以能"动手"操作真实的文件、网络，靠的是什么机制？
   - A. 模型自己长出了手
   - B. **工具调用（tool use）**：模型发出"用某个工具"的请求，外面的程序真去执行，再把结果喂回给模型
   - C. 模型把自己下载到了你电脑里
   - D. 纯靠模型记忆里的东西
   > **答案 B。** 模型本身只会生成文字（第4章），它通过"请求调用工具 → 程序真执行 → 结果喂回"这套机制间接地"动手"：**模型动脑、程序动手**。A、C 是对"动手"的字面误解；D 错在——动手恰恰是为了用上模型记忆**之外**的真实信息和操作。

3. **[基础·概念题]** Agent 的招牌工作方式，最贴切的描述是？
   - A. 你问一句、它答一句，问完就停
   - B. 给它一个目标，它自己跑"调查→动手→检查→调整"的循环，多步推进直到完成
   - C. 它把所有可能的答案一次全列出来
   - D. 它只负责出主意，做还是你做
   > **答案 B。** "自己跑循环、多步推进"正是 Agent 区别于聊天机器人的核心。A、D 描述的恰恰是**聊天式 AI**（你推一下它走一步、只出主意）；C 与 Agent 的工作方式无关。

4. **[进阶·误区题]** "Agent 能力强，能自己把活干完，所以可以放心地丢给它、撒手不管。"错在哪？
   - A. 没错，越强越该放手
   - B. 能力和风险是**同一枚硬币**：它能替你做事就能替你闯祸（改错、跑出意外后果），而且出错时常常不停下来问你——必须配许可/确认和可回滚环境
   - C. 它不会出错
   - D. 出错了它会自动撤销
   > **答案 B。** 这是本章最该记牢的一点：Agent"会动手"既是它的价值、也是它的风险来源。正因为它可能擅自做出你没料到的事、还自顾自走下去，所以要靠**许可机制 + 可回滚**来兜底（第26、27、40章；本地动手先用 Git，第24章）。A 危险；C、D 都把它想得太可靠了——它真会出错，也不会自动替你收拾。

5. **[基础·情景题]** 你让一个 Agent 去整理你电脑上一个重要文件夹（移动、改名、删重复）。动手前，最该先做的是什么？
   - A. 直接让它跑，相信它不会出错
   - B. 先把文件夹**备份**（或在能回滚的环境里操作），并对"删除、覆盖"这类关键动作要求它**先问过你**再做
   - C. 把电脑关机
   - D. 让它一次性全做完，最后再看结果
   > **答案 B。** Agent 会真的动手，**改错删错就是真的改错删错**——所以"可回滚（备份/Git）+ 关键动作先确认"是用 Agent 的基本安全姿势（第26、27、40章）。A、D 都是"撒手不管"，正是本章反复警告的危险用法；C 解决不了任何问题。

6. **[进阶·操作/观察题]** 找一个你能接触到的、带"动手"能力的 AI 工具（比如能联网搜索、能读你上传的文件、能调用工具的那种）。给它一个**需要好几步**的小任务，观察它是不是会"自己先查一下、再动手、再根据结果调整"。再对照一个**只会聊天**的 AI 做同样的事，体会两者的差别。
   > **该观察到：** 带工具的那个，会表现出"调查→动手→检查→调整"的循环——它会自己去查、去执行、根据中途结果改变做法，**往前走好几步**；只会聊天的那个，只能给你一段"你应该这么做"的文字，真正执行还得你来。亲手对比一次，你就摸到了本章的核心：Agent 的不同**不在更聪明，而在会真的动手**（也别忘了：会动手 = 要管好它，关键动作让它先问过你）。
