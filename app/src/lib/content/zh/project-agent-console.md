前面三个项目（第31、32、33章），AI 基本是"你说一句、它做一句"地陪着你干。这一章我们换个玩法：搭一个 **Agent 控制台**——给它一个目标和一套流程，让它**自己连着跑好几步**，你坐在旁边看它"调查→执行"，只在关键处出手。

这正是第19章那个"会动手的 AI"第一次真正落到你手上。也正因为它会动手，这一章的重头戏不是"怎么让它跑得更猛"，而是反过来——**先给它装好"刹车"**。你会发现，搭一个能放心用的 Agent，难的从来不是让它跑，而是想清楚**哪几步绝不能让它自己做主**。

> 〔本章讲的是"怎么带一个会自己跑的 Agent"这套通用做法，不绑定某个平台。具体在哪个产品、用什么界面搭、怎么配工具，各家不同且更新很快，一律**以官方文档为准**。〕

## 一、准备

动手前，先把"我到底要搭个什么"想明白，别上来就堆功能。

- **挑一个"多步、但每步都不危险"的小目标练手。** 第一次搭 Agent，目标要小、要安全。好例子：让它"读一个文件夹里的几份文档，各写一句话摘要，汇总成一张清单"——这需要好几步（逐个读、逐个总结、最后汇总），但**每一步都不删东西、不发东西、不花钱**。坏例子：让它"自动清理我的邮箱""自动回复客户"——第一次就碰这种不可逆的活，等于把新手司机直接扔上高速。
- **想清楚这件事拆成哪几步。** 你得先在脑子里（或纸上）把目标拆成"调查→做→检查"的几步。你自己都说不清的流程，没法交给它跑（这是第14章那个"万能循环"的实战版）。
- **准备好一个"能反悔"的环境。** Agent 会真的动手，所以**它能碰到的东西，最好都是能恢复的**。涉及本地文件，就在用 Git 管理的、单独的文件夹里试（呼应第24、27章）；用平台自带的 Agent，则看它提供的隔离/沙箱方式——**以官方文档为准**。
- **最重要的一件事：列一张"必停清单"。** 在让它跑之前，先把"哪几步必须我点头才能做"写下来。这张清单是这一章的灵魂，下面第二节会手把手教你列。

> **要点：** 搭 Agent 的准备工作，一半是技术（环境、工具），另一半是**判断**（目标够不够安全、流程拆没拆清、刹车列没列全）。后面这一半，才是真正决定你会不会翻车的地方。

## 二、一步步做

下面这套流程，不依赖任何具体产品。你在哪个平台搭，把这套思路套进去即可（具体按钮、命令**以官方文档为准**）。

### 第 1 步：用大白话给它一个目标 + 一套流程

别只丢一个目标就撒手。把"按什么步骤走"也一起交代清楚。一个面向非程序员的请求大概长这样：

```text
目标：把"待整理"文件夹里的几份文档，各写一句话摘要，汇总成一张清单。

请按这个流程走：
① 先调查：列出文件夹里有哪几份文档，先别动手，给我看一眼。
② 再逐个读、逐个写一句话摘要。
③ 最后汇总成一张清单交给我。

规则（很重要）：
- 你只能"读"和"生成清单"，不许删除、不许修改原文件、不许把任何东西发出去。
- 调查完先停下来给我看清单对不对，等我说"继续"再往下做。
- 任何拿不准的地方，别替我猜，停下来问我。
```

注意这段话里，**一半篇幅在说"不许做什么"和"在哪停"**。这不是啰嗦，这就是你给它装的刹车。

### 第 2 步：先列"必停清单"——这章的核心

在你按下"开始"之前，把下面这类动作**预先**标成"必须人工确认"。判断标准只有一句话：**这一步可逆吗？有没有后果？**

| 可以让它自己做的 | 必须停下来等你点头的 |
| --- | --- |
| 读文件、整理、归类、生成草稿、列清单 | **花钱**：调用按次收费的服务、下单、扣额度 |
| 在几个无关紧要的小选项里挑一个 | **删数据 / 覆盖**：删除、清空、改写你的原始文件 |
| 做完一步、汇报一下再做下一步 | **对外发送**：发邮件、提交、发布、对外调用接口 |
| 把内容从一种形式换成另一种 | **它拿不准、需要你拍板**的判断 |

这张表就是第14章"握着刹车"和第26章"许可机制"在项目里的落地。**列清单的时机很关键：要在它动手之前列，不是等它闯了祸再补。**

### 第 3 步：让它跑，你盯着"调查→执行"的过程

按下开始后，**别走开**。Agent 的价值是它自己往前跑，但你第一次用，要把它跑的每一步看在眼里：

- **看它的"调查"对不对**：它有没有找对文件、理解对目标？第一步就歪了，后面会在错的前提上越做越"工整"（第14章讲过这个坑）。
- **看它到了"必停点"会不会真的停**：跑到该确认的地方，它应该停下来问你。如果它没停、径直做了不可逆的动作——说明你的刹车没设好，立刻叫停。
- **觉得它跑偏，随时打断**：把话说清楚，再让它继续；或者干脆回滚重来。

### 看结果 / 验收

一轮跑完，按这三条验收（呼应第27章的"验收三问"）：

1. **它到底做了什么？** 不只看它给的最终清单，还要看它中途碰了哪些东西——有没有动到你没让它动的地方。
2. **该停的地方停了吗？** 回头确认：那几个"必停点"，它是不是都老老实实停下来等你了。这一条比结果漂不漂亮更重要。
3. **结果你能不能信？** 它生成的摘要，关键信息自己抽查几条——它的自检不可全信（第14章），漂亮不等于对。

三条都过，这次 Agent 才算"既跑通了，又没失控"。

## 三、常见坑

| 坑 | 怎么回事 / 怎么躲 |
| --- | --- |
| **一上来就给它"不可逆"的大活** | 第一次就让它删邮件、发消息、动钱——出事就是真出事。先用"只读不写"的安全目标练手 |
| **只给目标，不给刹车** | 光说"帮我把邮箱清理一下"，没说"删之前先问我"——它可能一路删到底。**刹车要和目标一起给，且写在它动手前** |
| **按下开始就走开** | Agent 会自己跑，但第一次用必须全程盯着，尤其盯它"到必停点会不会真停"。撒手不管正是第19章反复警告的危险用法 |
| **把"它说检查过了"当成真检查过** | 它的自检还是在"接话"，不是真有一双客观的眼睛（第14章）。关键信息自己再核 |
| **跑偏了还在乱局里硬纠** | 越纠越歪。干脆**回滚到干净状态、开个新对话**，把"刚才踩的坑"作为前提重讲（第13、30章） |
| **以为"步数越多 = 越强"** | 让它一口气跑十几步，你根本没法逐步把关。**步子迈小一点、节点设密一点**，反而更稳 |

## 四、可以再往前一步

把第一个安全的小 Agent 跑顺之后，可以这样**谨慎地**扩展——每一步都要重新问"刹车够不够":

- **给它接一两个工具或数据源**（用第22章的 MCP/Skills 思路），让它能读更多东西、做更多事。但记住下一章和第36章反复讲的：**每接一个新能力，就多一个出错和受攻击的入口**，要按需接、最小化接。
- **把你常跑的流程固化下来**：如果某套"调查→做→检查"你反复要用，可以把它打包成一个固定流程（Skills 的思路，第22、36章），下次一句话调用。
- **逐步放宽"必停点"——但要非常克制**：只有当你对某一步的安全性彻底有把握、且它真的可逆时，才考虑让它自己做。**涉及花钱、删数据、对外发送的那几条，建议永远留在"必停"里**，无论你多熟。
- **写一份"它该怎么干活"的说明**：把这个项目的规矩、雷区、它能碰和不能碰的东西写下来交给它（项目上下文的思路，第28章），让它跨会话都守规矩。

> **要点：** 搭 Agent 控制台，真正的功夫不在"让它能干多少事"，而在"**你提前想清楚了哪几步不能让它自己做主**"。能力是它带来的，安全是你设计出来的。一句话记一辈子：**先装刹车，再给油门——花钱、删数据、对外发送，这三类动作永远先问过你。**

## 小结

- 这个项目让你第一次真正带一个**会自己连跑多步的 Agent**（第19章），核心练的不是"让它跑猛"，而是"让它跑得住"。
- 准备阶段最关键的不是技术，是**判断**：挑一个"多步但每步都安全"的目标、把流程拆清、列一张"必停清单"、备好能回滚的环境。
- 做的时候：**用大白话给目标 + 流程 + 刹车**，盯着它"调查→执行"的过程，重点看它"到必停点会不会真停"。
- 验收三问：它到底做了什么、该停的地方停没停、结果能不能信。**该停没停，比结果漂不漂亮更要命。**
- **花钱、删数据、对外发送**这三类不可逆动作，建议永远设成"必须人工确认"（呼应第14、19章）。具体平台与搭法**以官方文档为准**。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。答案与解析在每题下方的引用块里——先自己想，再对照。

1. **[基础·概念题]** 搭一个 Agent 控制台，和前几个项目里"你问一句它答一句"最根本的区别是？
   - A. 界面更炫
   - B. 它能朝着一个目标**自己连着跑好几步**（调查→执行→检查），你只在关键处把关
   - C. 它用的模型更强
   - D. 它不会犯错了
   > **答案 B。** Agent 的要害是"会自己往前走一段"（第19章），而不是更聪明或更不会错。正因为它自己跑、还可能不停下来问你，这一章才把"装刹车"放在第一位。

2. **[基础·概念题]** 这一章反复强调的"刹车"（必停清单），本质是什么？
   - A. 让 Agent 跑得慢一点
   - B. **预先**把"花钱、删数据、对外发送、需要拍板"这类不可逆/有后果的步骤，标成"必须人工确认"
   - C. 一个让 Agent 关机的按钮
   - D. 限制 Agent 只能读不能想
   > **答案 B。** 刹车不是拖慢它，而是**圈定安全范围**：范围内放手跑，碰到不可逆/有后果的动作就必须停下等你点头（呼应第14、19章）。判断标准是一句话：这一步可逆吗？有后果吗？

3. **[基础·误区题]** "我把目标交代清楚、按下开始，就可以去忙别的了"——第一次搭 Agent 时，这个想法错在哪？
   - A. 没错，Agent 就是为了让你撒手
   - B. 第一次必须全程盯着，尤其要看它"到必停点会不会真的停"；撒手不管正是危险用法
   - C. 错在不该给它目标
   - D. 错在 Agent 根本不能自己跑
   > **答案 B。** Agent 会自己跑没错，但"它会不会在该停的地方真的停下来"得你亲眼确认过才放心。它没停、径直做了不可逆的事，说明刹车没设好，要立刻叫停（第19章）。彻底甩手是本书反复警告的危险姿势。

4. **[进阶·情景题]** 你让 Agent 帮你整理一个客户资料文件夹。下面哪一步**最该**被设成"必须停下来问你"？
   - A. 把资料按公司名排序
   - B. 把表格统一成一种格式
   - C. 把它认为"重复多余"的几条客户记录**直接删掉**
   - D. 把内容从表格整理成一段文字说明
   > **答案 C。** 删除是**不可逆**且有重大后果的动作，正是必停点。A、B、D 都是范围内可以放手的低风险整理。区分标准始终是：这一步可逆吗、有没有后果（呼应第14章）。

5. **[进阶·情景题]** Agent 跑到一半明显跑偏了，越做越乱。最佳止损动作是？
   - A. 在乱局里一句句追着它纠正
   - B. 回滚到干净状态、开一个新对话，把"刚才踩的坑"作为前提重新交代
   - C. 把电脑关机
   - D. 让它先把这一轮做完，最后再说
   > **答案 B。** 在被污染的上下文里硬纠，往往越纠越歪（第13、30章）。所以准备阶段才要备好"能回滚的环境"——就是为了这一刻能干净利落地重来。C 解决不了问题；D 是放任它在错的方向上越跑越远。

6. **[基础·操作/观察题]** 不必真搭平台——先挑一件你会让 Agent 做的多步小事（如"读三份文档各写一句话摘要再汇总"），把它拆成"调查→做→检查"几步，再为它列一张"必停清单"：哪几步必须你点头？（具体怎么搭**以官方文档为准**。）
   > **该观察到：** 你会发现，安全的练手目标里，"必停点"可能一个都没有（全是只读、不发、不删）——这正是第一次该挑的目标。一旦目标里出现"删、发、花钱"，对应的步骤就必须进"必停清单"。**先想清楚刹车，再谈让它跑**，这就是搭 Agent 的正确顺序（呼应第14、19章）。
