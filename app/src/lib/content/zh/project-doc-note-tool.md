每个人的电脑里都有那么一个"杂物抽屉"：一个塞满了零散文件的文件夹。几十篇会议记录、一堆下载来的文章、东一个西一个的笔记，文件名乱七八糟，想找一篇全靠翻。

这一章的项目，就是收拾这个抽屉。我们做一个**文档 / 笔记小工具**——其实不是一个花哨的软件，而是借编程智能体（coding agent）之手，把一堆零散文件**批量整理、转换、汇总**成你想要的样子。比如：把几十个文件按内容归类、统一改名、把它们的要点汇成一张总表、把一种格式批量转成另一种。

为什么用这个练手？因为它能让你亲身体会一件特别朴素、也特别强大的事：**AI + 文件 = 最省力的自动化**。你过去要一个一个手动干上一下午的重复活，交给它，说清楚规则，它能成批地、不知疲倦地做完。第 14 章讲过"让 AI 持续推进任务"，这一章就是它最接地气的一个落地。

但这一章有一条**别的项目没有的安全红线**，先撂在最前面，整章都要绷着这根弦——

> **要点：** **动手批量处理之前，先把原件备份一份。** 这是这一章的头号铁律，没有例外。原因很简单：上一章做网站，改坏了刷新一下就知道、退回去就行；可这一章它是在**成批地动你的真实文件**——一旦它误删、误改、批量改错了几十个文件，而你又没有备份，那就是真的找不回来了。**备份原件，是你做任何批量操作前必须先扣上的安全带。**

## 一、准备：需要什么、最小起点

**你需要的最小起点：**

- **一个编程智能体**：Claude Code 或 Codex 之类，它能读你文件夹里的文件、按你的话整理和改写。它有终端、IDE、网页任务、云端等多种形态，**有的在本地跑、有的在云端跑，具体形态各异、还在更新，以官方文档为准**（第 24 章讲过通用工作流）。
- **那个要整理的文件夹**：就是你那个"杂物抽屉"。
- **一份原件备份**（**最重要**）：在让它动手之前，**先把整个文件夹原样复制一份**放到别处，比如改名叫"`原件备份_勿动`"。这份是你的保命副本，全程不去碰它。
- **一个能反悔的安全网**：除了上面那份手动备份，**当它在你本地动手时**，最好也把工作文件夹用版本控制（Git）管起来（第 27 章），多一层保险。

> **要点：** 备份和"用 Git"是**两件事，都要做**，不是二选一。手动复制一份原件，是最朴素、最不会出错的保命招，连不懂 Git 的人都能做；Git 则让你能看清"它到底改了哪些文件、怎么改的"并随时回退。批量操作面对的是真实文件，**两层保险一起上，才睡得着觉**。

**心态准备**：还是那句话——把它当"能干但刚入职的同事"（第 24 章）。你让他整理几十份文件，他手脚很快，但**会不会理解错你的归类规则、会不会手抖删错文件，你得盯着**。所以这一章我们格外强调一件事：**先拿一两个文件试，确认它懂了规则、做对了，再放它去处理一整批。**

## 二、一步步做

下面每步给你一句**对 AI 说的大白话**（再强调：这是对 AI 说的话，**不是要你去敲的命令**），以及**这步人要确认什么**。注意整体节奏是：**备份 → 先小批试 → 验收无误 → 再上全量。**

### 第 0 步：先备份，再让它"摸清现状"

第一件事，永远是备份。

> 大白话请求：*"我准备整理这个文件夹里的文件。动手之前，请先提醒我：你不会动我另外那份'原件备份'对吧？我们只在工作文件夹里操作。"*

**人要确认什么：**
- 在它做任何整理动作**之前**，你自己**先手动把整个文件夹复制一份**当原件备份。这一步靠你，不靠它。
- 然后让它先**只看、不动**，摸清现状：

> 大白话请求：*"先别动任何文件，只帮我看看这个文件夹里大概有些什么：一共多少个文件、都是什么类型、文件名有没有什么规律。给我一个清单。"*

**人要确认什么：**
- 看它给的清单，确认它**确实看懂了你这堆文件的现状**（数量对不对、类型有没有看漏）。这一步是"调查"（第 14、24 章的工作流第一步），让它先了解、你先核对，能避免它一上来就瞎整。

### 第 1 步：把"整理规则"说清楚

整理这件事，成败全在"规则说得清不清楚"。含糊的规则，它只能瞎猜。

> 大白话请求：*"我想这样归类：文件名里带'会议'两个字的，归到'会议记录'文件夹；带'报销''发票'的归到'财务'文件夹；剩下的归到'其他'。归类的时候**只是移动文件，不要改文件内容**。"*

**人要确认什么：**
- 规则里的**模糊地带**要替它想到：一个文件名既带"会议"又带"发票"怎么办？没有任何关键词的算"其他"吗？**你想清楚了告诉它**，别让它自己猜。
- 特别留意那句"**只移动、不改内容**"——把"它能动什么、不能动什么"讲死，是这一章最该养成的习惯。

### 第 2 步：先拿一小批试，看它懂没懂

**这一步是这一章的灵魂，千万别跳。** 规则说完，**先别放它整一整批**，让它**只处理两三个文件做示范**。

> 大白话请求：*"先别动全部。挑 3 个文件，按刚才的规则演示一遍你会怎么归类，告诉我每个文件你打算放进哪个文件夹、为什么。先别真的移动，说给我听。"*

**人要确认什么：**
- 看它对这 3 个的判断**符不符合你的本意**。符合，说明它懂了规则，可以往下走；不符合，**就地把规则补清楚**，再让它重说一遍——**这时候纠正，代价极小**。
- 想象一下反例：要是跳过这步、直接让它整几十个，等它按"理解错的规则"把全部归错位，你再发现，那就是几十个文件的烂摊子。**先小批试，就是用三个文件的成本，挡掉几十个文件的灾难。**

### 第 3 步：确认无误，再上全量

小批试通过了，才放它处理整批。

> 大白话请求：*"判断都对。现在按这个规则，把整个文件夹的文件都归类好。"*

**人要确认什么：**
- 它做完后，**抽查几个**：随便点开几个文件夹，看里面的文件是不是该在的都在、不该在的没混进来。
- 如果用了 Git，可以让它**说一下"这次一共动了哪些文件、移动到了哪里"**，对照着看一眼（diff，第 25 章）。看不懂技术细节没关系，重点是确认"动的范围和你预期的一致"。
- **万一发现整批都错了**：别慌，也别在错的基础上接着改——这正是开头那份备份的用武之地，**退回去、重来一遍**。

### 第 4 步：进阶玩法——汇总和转换

归类只是开胃菜。"AI + 文件"真正省力的地方，在**汇总**和**转换**。

> 大白话请求（汇总）：*"把'会议记录'文件夹里每个文件的标题、日期、最重要的一条结论，提取出来汇成一张表，存成一个新文件。**原来的文件别动。**"*

> 大白话请求（转换）：*"把这个文件夹里的文本笔记，统一转换成同一种格式，转换结果放到一个新的'已转换'文件夹里，**原文件保留不动。**"*

**人要确认什么：**
- 汇总表做好后，**挑两三行回到原文件核对**——它提取的标题、日期、结论是不是真的、有没有抄错或编造（它会"脑补"，第 9 章）。**汇总最容易出的错就是悄悄改了数字或编了结论**，必须抽查。
- 注意上面两句都强调了"**结果放新文件 / 新文件夹，原件不动**"。这是个好习惯：**让它产出新东西，而不是覆盖旧东西**，这样原件永远安全。

## 三、常见坑

- **跳过"先小批试"，直接整全量。** 这是这一章最致命的坑。规则只要它理解偏一点，全量跑下来就是成批的错。**永远先拿两三个试，确认无误再放量。**
- **没备份就动手。** 重说一遍开头的红线：它动的是你的**真实文件**，误删误改没备份就找不回。**备份是动手前必扣的安全带，没有例外。**
- **规则给得太含糊。** "把乱的文件整理一下"——什么叫乱、整理成什么样，它只能猜，猜错怪不得它。**规则越具体（带什么字归到哪、模糊情况怎么办），它做得越准。**
- **让它"原地覆盖"而不是"另存新文件"。** 转换、汇总时如果让它直接改原文件，一旦出错原件就毁了。**尽量让结果落到新文件/新文件夹**，原件留着。
- **汇总结果照单全收，不核对。** 它提取信息时可能抄错、漏掉、甚至编造。**汇总表至少抽查几行回原文核对**，别把它的"脑补"当成真数据用出去。
- **把含私密信息的文件一股脑丢给它。** 文件里如果有身份证号、密码、客户隐私这类东西，**那是会被发出去的**（第 37 章）。处理敏感文件前先想清楚什么能交、什么得先抹掉或干脆别交。

## 四、可以再往前一步

整理、汇总、转换跑顺了，可以试试更"自动化"一点的玩法（都还是"说清规则 + 先小批试 + 核对"的老套路）：

- **统一改名**：*"把这些文件按'日期_主题'的格式统一重命名，先拿 3 个演示给我看。"*
- **批量提取要点成目录**：*"给这一整个文件夹生成一个目录文件，列出每个文件的标题和一句话摘要。"*
- **找重复和过期**：*"帮我找出内容几乎重复的文件，列个清单给我，**先别删**，让我自己决定。"*（凡涉及"删除"，更要让它先列清单、由你拍板，绝不让它自动删。）
- **把这套规则变成"以后能重复用"的小流程**：你可以让它把整理逻辑整理成一份说明，下次新文件进来照着办。要不要做成更正式的自动化，量力而行，**具体怎么实现以官方文档为准**。

> **要点：** "再往前一步"里有一条不变的安全准则：**凡是会"删"或"覆盖"的操作，都让它先列清单、停下来等你点头，绝不让它自动执行。** 整理可以放手交给它，但"动不可逆的操作"这道决定权，**永远攥在你手里**（这也呼应第 26 章"许可机制"的精神：让它动手，但有后果的动作要你批准）。

## 小结

- 第二个项目体会"**AI + 文件 = 最省力的自动化**"：把一堆零散文件**批量整理、转换、汇总**成你要的样子。
- **头号红线：动手批量处理前，先把原件备份一份。** 它动的是真实文件，误删误改没备份就找不回——备份是必扣的安全带。
- 核心节奏是 **备份 → 先小批试（两三个文件确认它懂了规则）→ 验收无误 → 再上全量**；"先小批试"这一步是灵魂，千万别跳。
- **规则要说具体**（带什么字归哪、模糊情况怎么办），让结果**落到新文件/新文件夹、原件不动**；汇总表要**抽查回原文核对**，防它编造。
- 凡"删除/覆盖"等不可逆操作，**让它先列清单、由你拍板**，绝不自动执行（第 26 章）；含私密信息的文件交出去前先想清楚（第 37 章）。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。项目章侧重情景与操作。先自己想，再看引用块里的答案解析。

1. **[基础·概念题]** 这一章反复强调的"头号红线"是什么？
   - A. 一定要会写代码
   - B. 动手批量处理之前，先把原件备份一份
   - C. 必须联网才能整理文件
   - D. 文件越多越好
   > **答案 B。** 和做网站不同，这一章它动的是你的**真实文件**——误删、误改、批量改错而又没备份，就真的找不回了。备份是做任何批量操作前必须先扣上的安全带，没有例外。

2. **[进阶·情景题]** 你把归类规则告诉它之后，**最该先做什么**，再放它处理整批文件？
   - A. 直接让它把几十个文件全部归类好
   - B. 先让它拿两三个文件演示一遍，确认它的判断符合你本意，再上全量
   - C. 关掉电脑等一晚上
   - D. 把规则再含糊地说一遍
   > **答案 B。** "先小批试"是这一章的灵魂。用三个文件的成本，验证它有没有理解对规则；理解偏了就地补清楚，代价极小。跳过这步直接整全量，等于赌它一次就懂——错了就是几十个文件的烂摊子。

3. **[基础·误区题]** "我让它把转换结果直接覆盖原文件，省得文件多"——这个做法的风险是什么？
   - A. 没风险，更整洁
   - B. 一旦转换出错，原件就被毁了、找不回；应该让结果落到新文件/新文件夹，原件保留
   - C. 覆盖会更快
   - D. 原文件会自动变成两份
   > **答案 B。** 让它产出**新东西**而不是覆盖旧东西，原件就永远安全。"另存新文件、原件不动"是这一章贯穿始终的好习惯。

4. **[进阶·情景题]** 它做好了一张"会议要点汇总表"。在你把这张表发给同事之前，最该做的一步是？
   - A. 直接转发，它做的肯定对
   - B. 抽查两三行，回到原文件核对标题、日期、结论是不是真的，有没有抄错或编造
   - C. 把表删掉重做十遍
   - D. 让它再生成九张一样的表
   > **答案 B。** 它提取信息时可能抄错、漏掉、甚至编造（第 9 章）。汇总最容易出的错就是悄悄改了数字或编了结论——**至少抽查几行回原文核对**，别把"脑补"当真数据用出去。

5. **[进阶·情景题]** 你让它"找出内容重复的文件"，下一步你希望它怎么做最稳妥？
   - A. 让它发现重复就自动全删了
   - B. 让它先列一个清单给你，**先别删**，由你逐一决定
   - C. 让它把重复的覆盖成一个
   - D. 不用管，随它处理
   > **答案 B。** 凡是会"删"或"覆盖"的不可逆操作，都该让它先列清单、停下来等你点头（呼应第 26 章"许可机制"）。整理可以交给它，但"动不可逆操作"的决定权永远在你手里。

6. **[基础·操作题]** 在让 AI 整理你某个真实文件夹之前，请先**亲手把整个文件夹原样复制一份**，改名为"原件备份_勿动"放到别处。做完回答：如果之后它把工作文件夹整得一团糟，你靠什么把东西找回来？
   > **该观察到：** 你靠的就是这份手动备份——它是连不懂 Git 的人都能用的、最朴素的保命招。再加上把工作文件夹用 Git 管起来（能看清它改了什么、随时回退），就是"两层保险"。批量操作面对真实文件，两层一起上才安心。具体工具怎么用**以官方文档为准**（第 25、27 章）。
