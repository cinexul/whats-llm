第2章我们破除了一个大误会：大模型不是数据库，它的回答是"现算生成"的，不是查出来的。可一个新问题立刻冒出来——**既然不是查的，那它到底怎么把一段通顺的话"算"出来的？**

答案出人意料地简单。简单到很多人第一次听会愣住：**它从头到尾，只在反复做一件事——预测"接下来最可能出现的下一个词（token）"。** 没有"先想好答案",没有"打好腹稿",就是一个词一个词地往外蹦，蹦的时候只问一句："按前面这些，下一个词最该是啥？"

这一章把这件"唯一的事"讲透。理解它，你会一口气想通三件事：为什么它能写出长文、代码、表格；为什么同一个问题它每次答得不太一样；以及最关键的——**为什么它注定会"一本正经地编"**（这是第9章幻觉的根）。

## 一、一句话原理：它是放大了亿万倍的"输入法联想"

你每天都在用一个"会预测下一个词"的东西——**手机输入法**。

你打"今天天气真",输入法上面就跳出候选："好""不错""热"……它凭什么猜？因为它统计过海量文字，知道"今天天气真"后面**最常跟**哪些词。

**大模型干的，本质就是同一件事，只是放大了亿万倍。** 你给它一段开头，它就根据训练学到的语言规律（第2章那套"拧好的旋钮"）,算出"下一个 token 最可能是什么",挑一个接上；然后把"开头+刚接的这个"当成新的开头，**再算下一个**;如此一个接一个，**滚雪球一样**滚出一整段话。

```text
它每一步都在做这道"填空题":

  前面已有: "中国的首都是"
       ↓  算一算,下一个 token 最可能是?
  接上一个: "北"
       ↓  把"中国的首都是北"当成新开头,再算
  接上一个: "京"
       ↓  继续……
  最终滚出: "中国的首都是北京。"
```

> **要点：** 它**不是先想好"北京"这个答案再写出来**,而是一步步"猜下一个 token、接上、再猜",像滚雪球一样滚出来的。所谓"它在回答问题",拆开看，**全程只是亿万次"猜下一个词"的接力**。这就是大模型的核心，没有更玄的了。（手机输入法和它的差距，主要在于它学的规律复杂得多、记得住的"前文"长得多——长到第3章说的整个上下文窗口。）

## 二、它怎么"挑"下一个词：一张概率清单

说"预测下一个词",更准确的画面是：每走一步，模型会给词表里**每一个候选 token 都打一个概率分**,排成一张清单，然后从中**挑一个**接上。

举个例子。假设前面是"日本的首都是",模型内部算出来的清单大概长这样：

```text
前文:"日本的首都是" → 下一个 token 的概率清单(数字为虚构示意):

  东京 ........ 92%   ← 概率最高
  日本 ........  2%
  大阪 ........  1.5%
  江户 ........  0.8%
  ……(剩下成千上万个候选,概率都很低)
```

> **重要提醒：** 上面这些百分比是**为了说明而虚构的例子**,不是任何真实模型的真实数值——真实概率是模型内部算出来的、你平时看不到的。这里只用它帮你建立一个画面：**模型眼里的"下一个词",是一张带概率的候选清单，而不是唯一确定的答案。**

挑词时通常**偏向高概率的**,但**不总是死挑第一名**。这里藏着一个你早就察觉过的现象——

- 如果每次都**雷打不动挑概率最高的那个**,那同一个问题问十次，答案会一模一样、刻板。
- 实际上模型在挑的时候带一点**可控的随机性**:有时也会挑中概率第二、第三的词。于是同一个问题，这次和下次的措辞就**不完全一样**。这也解释了第2章那个小实验——同样的问题问三遍，用词总有出入。

> **实际会怎么遇到：** 你对某个回答不满意，点"重新生成"或换种问法，常能得到一个更好的版本——靠的就是这点随机性，等于让它**换一条"挑词路径"重走一遍**（第11章会讲"换个说法"的技巧）。这点随机性具体怎么调（那个叫"温度"的旋钮）,我们留到**第8章**专门讲。你现在只要记住：**它的下一个词是从一张概率清单里"挑"出来的，挑的过程带一点随机，所以输出会变。**

## 三、就这一条，凭什么能写长文、代码、表格？

你可能会怀疑：不就是"猜下一个词"嘛，这么朴素的动作，怎么写得出逻辑通顺的长文章、能跑的代码、对齐的表格？

关键在于：**"下一个词"是带着前文一起算的**。它每猜一个词，都把**前面所有内容**（你的问题 + 它已经写出的部分）通通纳入考量（这正是第3章"上下文窗口"装着的东西）。于是：

- 写**文章**:它前面写了"第一，……",算下一段时，"第二，"就成了高概率的接续——**结构就这么自然冒出来了**,不需要谁规定。
- 写**代码**:代码也是一种"语言",有极强的规律(括号要配对、缩进有讲究、写了 `if` 后面常跟条件)。它把这些规律也学进了参数，于是**一个 token 一个 token 地，也能把代码"接"得合乎语法**。
- 列**表格**:Markdown 表格的 `|` 和 `---` 也是有套路的文字模式，它照样能按规律续出来。

```text
为什么"只猜下一个词"却能成篇:
  每一步都带着"前面全部"一起猜
  → 上一句决定了下一句的走向
  → 一步步接力,逻辑和结构就"长"了出来
```

> **要点：** 别小看"猜下一个词"。因为**每一步都被前文牢牢约束**,这些"局部最合理的下一个词"接力下来，整体就呈现出**像是有规划、有逻辑**的样子。但请记住：这是**自然涌现的连贯**,不是它"先有蓝图再施工"。这个区别，马上引出本章最重要的一点。

## 四、最该记住的一点：它追求"像",不追求"对"

现在，把前面三节合起来，会推出一个让人有点不安、但极其重要的结论。

模型每一步挑的，是**"最可能、最自然、最像那么回事"的下一个词**——它的全部目标是让文字**通顺、像样、符合语言习惯**。**它并没有一个"核对事实"的步骤**:它不会停下来想"我这么说是真的吗？有依据吗？"它只关心"接下来这么写，像不像一段正常的话"。

换句话说：

> **大模型的内在追求是"听起来对（像真的）",而不是"事实上对（是真的）"。**

这一条直接解释了它最让人头疼的毛病——**它会一本正经地胡说（幻觉）**:

- 你问一个它"印象模糊"的事实，它**不会**说"我不确定"（那不是它的工作方式）,而会**照样顺出一段最像答案的文字**——哪怕是编的。因为对它来说，"编一个像样的答案"和"说出真答案",在"猜下一个词"这台机器里，**做的是同一件事**:都是挑最可能的下一个词往下接。
- 越是它该流畅的地方（格式、语气、措辞）,它越能编得**天衣无缝**。所以**越自信、越流畅的回答，越不能想当然地信**——通顺只代表"像",不代表"对"。

> **常见误解：** "它都把答案讲得头头是道了，肯定是先理解了我的问题、想清楚了才回答的吧？" —— **顺序反了。** 它不是"先理解、再组织、最后表达";它是**一边猜下一个词、一边把话生出来**,连贯和"看起来懂"是这个过程的**副产品**,不是前提。把这点想透，你就明白为什么它能答得很顺、却又会在事实上翻车——**流畅和正确，在它这里本就不是一回事**（详见第9章幻觉）。

> **实际会怎么遇到：** 让它总结你贴进去的资料，它一般很靠谱（因为答案就在眼前的上下文里，它"接"的就是你给的内容）;但让它凭空报一个数据、一条出处、一段引文，就容易出错——因为这时它没有可"接"的依据，只能**按"这种答案通常长什么样"去编一个像的**。记住这条分界，你就知道**什么时候能放心用、什么时候必须复核**（第15章）。

## 五、常见误解，集中纠正

| 误解 | 实际 |
| --- | --- |
| 它先在脑子里想好整个答案，再写出来 | 它**没有事先的答案**,是一个 token 一个 token **边猜边写**滚出来的 |
| 它的回答是从某处"查"出来的 | 它是**按概率现场生成**的（承接第2章：不是数据库） |
| 同样的问题，它每次答案应该一字不差 | 挑词带一点**随机性**,所以措辞每次常不同（随机性怎么调见第8章） |
| 它讲得通顺，说明它"理解"并"核对"过了 | 通顺只代表"像一段正常的话";它**不核对事实**,追求的是"像"不是"对" |
| 既然会"猜下一个词",那它只能写短句 | 因为每步都带着**全部前文**一起猜，接力下来就能成篇、成文、成代码 |
| 它一本正经说错是"出 bug 了" | 这是"只求像、不求对"机制的**正常产物**（幻觉，第9章）,不是故障 |

## 小结

- 大模型的核心，从头到尾**只有一件事**:根据前文，预测**下一个最可能的 token**,挑一个接上，如此**接力滚出**整段话——像放大了亿万倍的输入法联想。
- 它挑词是从一张**带概率的候选清单**里挑的（本章百分比为虚构示意）;挑的过程带**一点随机**,所以同一问题的措辞每次会变（第8章讲怎么调）。
- 它能写长文、代码、表格，是因为**每一步都被全部前文约束**,局部最合理的词接力起来，**连贯与结构自然涌现**——不是"先有蓝图再施工"。
- 最要命也最要记的一点：它追求**"听起来对（像）",不追求"事实上对（对）"**,中间**没有核对事实这一步**——这正是它会**自信地编（幻觉）**的根源（第9章）。
- 由此得到一条实用分界：**有依据可"接"时（如总结你给的资料）较可靠，凭空生成硬事实时易出错，必须复核**（第15章）。

下一章（第5章）,我们换个角度看它的"内功":它怎么把词和词的**意思**变成可计算的"坐标",从而"感觉到"哪些词意思相近——这就是**向量与 embedding**,也是搜索、RAG（第20章）的基础。

## 小测验

> 共 6 题，涵盖概念、误区、情景、操作四类。答案与解析在每题下方的引用块里——先自己想，再对照。

1. **[基础·概念题]** 用一句话概括大模型生成文字的核心机制，哪个最准？
   - A. 它先想好完整答案，再一次性输出
   - B. 它根据前面的内容，反复预测"下一个最可能的 token",一个个接力滚出整段话
   - C. 它从数据库里检索出现成的句子
   - D. 它把你的问题翻译成命令去执行
   > **答案 B。** "预测下一个 token、接上、再预测"就是它的全部核心。A 把它想象成"先有腹稿"（顺序反了）;C 是"数据库"误区（第2章已破）;D 答非所问。记住：连贯是这个接力过程的副产品，不是前提。

2. **[基础·概念题]** "概率分布（那张候选词清单）"在生成时起什么作用？
   - A. 它是模型的开机界面
   - B. 每走一步，模型给各个候选 token 打概率分，再从中挑一个接上
   - C. 它记录你历史聊天的概率
   - D. 它决定网速
   > **答案 B。** 每一步，模型眼里的"下一个词"都是一张带概率的清单，挑一个接上。A、C、D 都与机制无关。补充：本章里"东京 92%"这类数字是**虚构示意**,真实概率你平时看不到。

3. **[进阶·误区题]** "它回答得条理这么清楚，一定是先理解了我的问题、核对清楚才说的。"这个想法错在哪？
   - A. 没错，它先理解再回答
   - B. 它是**边猜下一个词边把话生出来**,连贯和"看起来懂"是副产品；它**不核对事实**,追求的是"像"不是"对"
   - C. 它理解了，但懒得核对
   - D. 它先核对，后理解
   > **答案 B。** 顺序反了：不是"先理解、再表达",而是"边猜边写"。它没有"核对事实"这一步，所以能讲得很顺却在事实上翻车。这正是它会"一本正经地编"的根（第9章）。C 误以为它有意愿核对却偷懒（它根本没这个步骤）;D 是文字游戏。

4. **[进阶·误区题]** "它会一本正经地说错（幻觉）,肯定是程序出 bug 了。"哪里不对？
   - A. 对，这是 bug
   - B. 这恰恰是"只求像、不求对"机制的**正常产物**:没把握时它也照样顺出一个最像答案的东西，不是故障
   - C. 是网络断了
   - D. 是你问得太难，把它问坏了
   > **答案 B。** 幻觉不是 bug,而是"追求像、不核对事实"的**必然副产品**（第9章）。把它当 bug 的人，会期待"修好了就不犯"——但只要它还是"猜下一个词"的机器，就得靠**你复核**来防（第15章）。C、D 都没抓住机制。

5. **[基础·情景题]** 下面两件事，哪件更容易触发它"编造"、更需要你复核？
   - A. 让它把你刚贴进去的一篇文章总结成三点
   - B. 让它凭空报出某本冷门书的 ISBN 号、某篇论文的确切出处
   - C. 两件事一样安全
   - D. A 更危险
   > **答案 B。** 总结你给的资料时，答案就在眼前的上下文里，它"接"的是你提供的内容，相对可靠；而凭空报硬事实（编号、出处）时它**没有可接的依据，只能按"这种答案长什么样"去编一个像的**,最易出错。记住这条分界：**有据可接 → 较稳；凭空生成硬事实 → 必查**（第15章）。

6. **[基础·操作/观察题]** 找一个带"重新生成"的 AI,对**同一个开放问题**（如"给我的猫起三个名字"）连点几次"重新生成";再换一个**有唯一正确答案**的问题（如"3 加 5 等于几"）也试几次。对比：哪类问题的多次回答**变化大**,哪类**几乎不变**?
   > **该观察到：** 开放问题（起名）每次**变化明显**——因为多个候选词概率都不低，挑词的随机性就有发挥空间；而"3+5"这种答案高度确定，"8"的概率压倒性地高，几乎每次都一样。亲手感受这点随机性，你就理解了"它从概率清单里挑词、挑得带点随机",以及为什么"换个说法/重新生成"常能帮你拿到更好的版本（随机性怎么调，见第8章）。
