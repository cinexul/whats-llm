欢迎翻开这本书。在正式开始前，请先花三分钟读完这一页——它会告诉你**这本书想带你去哪里、你该怎么读、以及一个能让你少走弯路的心态**。

这本书面向**懂基本电脑操作、但没有 AI 或编程背景的普通人**。如果你能收发邮件、会用浏览器和办公软件，你就完全能读懂它。**你不需要会编程**；遇到要"动手做项目"的章节，我们也会带你用大白话指挥 AI，而不是让你去背命令。

## 一、这本书想带你去哪里

读完它，你会获得四样东西，由浅入深：

| | 你将学会 | 对应部分 |
| --- | --- | --- |
| ① 看懂 | 大模型到底是什么、为什么能接话、为什么会说错——不再被一堆名词唬住 | 第一部分 |
| ② 会用 | 把 AI 用到日常、学习、工作里，问得准、喂得对、不被带偏 | 第二部分 |
| ③ 看清 | 模型 / 产品 / API / Agent、RAG、本地模型……一张地图认清整个工具生态 | 第三部分 |
| ④ 会做 | 用 **Claude Code 和 Codex** 这两个主流工具，做出你自己的项目 | 第四、五部分 |

最后还有第六部分，讲**风险与边界**——什么能喂给 AI、什么绝对不能，怎么不烧钱、不踩雷、负责任地用。

> **要点：** 这不是一本"AI 使用技巧大全"，而是想让你**真正理解 AI 是怎么回事**，从而自己判断"什么时候能信它、什么时候得自己把关"。理解了原理，技巧自然就有了。

## 二、两种读法，按需取用

你不一定要从头读到尾。挑一条适合你的线：

- **只想"会用"**：读**第一、二部分**（基础 + 日常使用），再跳到**第六部分**（风险）。这就够你把 AI 用得又顺又稳。
- **想"动手做东西"**：在上面的基础上，继续读**第三、四、五部分**（工具生态 → 编程实战 → 六个实战项目），一步步做出个人网站、文档工具、会议纪要助手，甚至给自己的程序接上 AI。

无论哪条线，建议**第一部分别跳**——它是整本书的地基，后面所有"为什么这样用"都建在它上面。

## 三、配套怎么用

- **每章末尾的「小测验」**：6 道题，分**概念题、误区题、情景题、操作题**四类。答案和解析就在题目下方——别只看对错，解析里讲的"为什么错、像谁会犯的错"才是重点。
- **术语表（附录 A）**：中日英三语对照，看到不懂的词随时来查。
- **提示词模板（附录 B）**：日常、工作、学习、编程等场景的"拿来即填"模板。
- **中日双语**：本书有中文版和日文版，点阅读器顶部的「**中文 / 日本語**」即可一键切换，内容一一对应。正文里关键术语**首次出现会用括号附上英文**（如 上下文窗口（context window）），方便你将来查英文资料。
- 阅读器还支持**左侧目录跳转、全文搜索、给文字划重点和写笔记**，读到哪、标过什么都会自动保存。

## 四、一个贯穿全书的心态

如果这本书只能让你记住两句话，那就是这两句——

1. **AI 很能干，但会"一本正经地说错"。** 它追求的是"听起来像真的"，不是"确实是真的"。所以**重要的事实、数字、引文，永远自己复核**。把它当一个聪明、好用、但偶尔信口开河的助手，而不是一本永远正确的百科全书。
2. **它默认"没有记忆"。** 你看到的"它记得我"，绝大多数是**产品层每次帮你把信息重新喂回去**，而不是模型本体真的记住了（这一点第 3、9 章会讲透）。理解了这条，很多"它怎么忘了""它怎么又记得"的困惑就都解开了。

> **要点：** 带着"它会错、它没记忆"这两点去读后面每一章，你会发现全书的方法和提醒，几乎都是在教你如何与这两条性质共处、并把 AI 用好用稳。

## 小结

- 这是一本面向普通人的**大模型基础 + AI 工具实战**手册，**不需要编程基础**。
- 四个目标：**看懂 → 会用 → 看清工具生态 → 用 Claude Code 和 Codex 做出项目**，最后守住风险边界。
- 两种读法：只想会用走"第一、二、六部分"；想动手做继续"第三、四、五部分"。
- 善用每章**小测验**、**术语表**、**提示词模板**，以及顶部的**中日双语切换**。
- 全程记住两句话：**AI 会自信地说错（重要的自己核）**；**它默认没有记忆（"它记得"多是产品层重新喂回）**。
